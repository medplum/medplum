# Spaces Bots

AI-powered Medplum bots that work together to provide a natural language interface for querying, summarizing, and visualizing FHIR data. These bots are designed to be used in conjunction with a Medplum Spaces application.

## Prerequisites

All bots require an OpenAI API key stored as a project secret:

| Secret | Description |
|---|---|
| `OPENAI_API_KEY` | OpenAI API key with access to GPT-4 models |

## Bots

### 1. FHIR Translator Bot (`fhir-translator-bot.ts`)

Converts natural language requests into FHIR R4 API calls using OpenAI function calling.

**How it works:**
- Receives a conversation history (`messages`) as a FHIR `Parameters` resource
- Uses OpenAI's function calling to produce a `fhir_request` tool call specifying the FHIR operation to perform (method, path, optional body)
- Returns the AI response along with a `visualize` boolean flag — set to `true` when the request would benefit from a chart (e.g., trends, growth charts, lab results over time)

**Input parameters:**

| Parameter | Type | Required | Description |
|---|---|---|---|
| `messages` | `valueString` (JSON) | Yes | Conversation history as a JSON array of `ChatMessage` objects |
| `model` | `valueString` | No | OpenAI model to use (default: `gpt-4`) |

**Output parameters:**

| Parameter | Type | Description |
|---|---|---|
| `tool_calls` | `valueString` (JSON) | The `fhir_request` tool call(s) generated by the AI |
| `visualize` | `valueBoolean` | Whether the result should be rendered as a chart |

**Example natural language → tool call mappings:**
- `"Find patient John"` → `GET Patient?name=John`
- `"Show growth chart for patient"` → `GET Observation?subject=Patient/{id}&code=body-height,body-weight` + `visualize: true`
- `"Create a follow-up task"` → `POST Task` with a Task resource body
- `"Update patient's phone number"` → `GET Patient/{id}` then `PUT Patient/{id}`

### 2. FHIR Summary Bot (`fhir-summary-bot.ts`)

Translates raw FHIR bundle responses into clear, human-readable summaries.

**How it works:**
- Receives a conversation history containing FHIR response data
- Uses OpenAI to produce a concise natural language summary of the FHIR resources
- Supports **streaming** via Server-Sent Events (SSE) when `responseStream` is available, or falls back to a standard non-streaming response

**Input parameters:**

| Parameter | Type | Required | Description |
|---|---|---|---|
| `messages` | `valueString` (JSON) | Yes | Conversation history including the FHIR response to summarize |
| `model` | `valueString` | No | OpenAI model to use (default: `gpt-4`) |

**Output:**
- Streaming: SSE stream with the AI-generated summary text
- Non-streaming: `Parameters` resource with the AI response

**Summary behavior:**
- Patient data → demographics, identifiers, key attributes
- Observations → values, dates, clinical significance
- Bundles → count and key details of results
- Empty bundles → friendly "no results" message
- Errors → plain language explanation with suggested next steps

---

### 3. FHIR Visualizer Bot (`fhir-visualizer-bot.ts`)

Generates self-contained React chart components from FHIR data using Recharts and Mantine.

**How it works:**
- Receives FHIR data and a conversation history
- Uses OpenAI to generate a `function Chart()` React component with hardcoded data derived from the FHIR resources
- Supports **streaming** via SSE when `responseStream` is available
- The generated component uses only pre-scoped Recharts and Mantine components — no imports required

**Input parameters:**

| Parameter | Type | Required | Description |
|---|---|---|---|
| `messages` | `valueString` (JSON) | Yes | Conversation history |
| `fhirData` | `valueString` (JSON) | No | Array of FHIR resources to visualize |
| `model` | `valueString` | No | OpenAI model to use (default: `gpt-4o`) |

**Output:**
- Streaming: SSE stream delivering a ` ```jsx ` code block containing the `Chart` component
- Non-streaming: `Parameters` resource with the generated code

**Chart type selection:**
- Numeric data over time → `LineChart` or `AreaChart`
- Categorical counts (e.g., task statuses) → `BarChart` or `PieChart`
- Non-numeric lists → `Table`
- Growth data (height/weight) → dual-axis `LineChart`

**Available components in generated code scope:**

| Library | Components |
|---|---|
| Recharts | `LineChart`, `BarChart`, `AreaChart`, `PieChart`, `ScatterChart`, `ComposedChart`, `Line`, `Bar`, `Area`, `Pie`, `Cell`, `XAxis`, `YAxis`, `CartesianGrid`, `Tooltip`, `Legend`, `ResponsiveContainer`, `ReferenceLine`, `Scatter` |
| Mantine | `Card`, `Title`, `Text`, `Group`, `Stack`, `Paper`, `Badge`, `Table`, `SimpleGrid`, `Box`, `Flex`, `ThemeIcon` |

## Typical Usage Flow

The three bots are designed to work in a pipeline:

```
User message
     │
     ▼
fhir-translator-bot   →   fhir_request tool call + visualize flag
     │
     ▼
[Client executes FHIR request against Medplum]
     │
     ├─ visualize: false ──▶  fhir-summary-bot   →  human-readable text
     │
     └─ visualize: true  ──▶  fhir-visualizer-bot →  JSX Chart component
```

1. **Translator** converts the user's natural language into a FHIR API call and signals whether a chart is needed.
2. The client executes the FHIR request and receives the data.
3. **Summary bot** narrates the results as plain text, or **Visualizer bot** renders the data as an interactive chart.
