# Multi-Brand Architecture

## Overview

The MEDrecord ecosystem consists of multiple brands, each serving different healthcare use cases while sharing common infrastructure and codebase.

| Brand | Purpose | Primary Users |
|-------|---------|---------------|
| **HealthTalk** | Patient questionnaires, voice/chat intake | Patients, Practitioners |
| **Coachi** | Coaching and mental health support | Coaches, Clients |
| **MedSafe** | Medication safety and compliance | Pharmacists, Patients |
| **MEDrecord** | Core EHR functionality | All healthcare roles |

For tenant (healthcare institution) isolation, see [Multi-Tenant Architecture](./multi-tenant.mdx).

## Brand Theming

### CSS Theme Files

Each brand has a theme file in `mrd/packages/ui/src/themes/`:

```css
/* healthtalk.css */
:root {
  --primary: oklch(0.55 0.2 250);      /* Blue */
  --primary-foreground: oklch(0.98 0 0);
  --accent: oklch(0.92 0.05 250);
}
```

```css
/* coachi.css */
:root {
  --primary: oklch(0.55 0.18 145);     /* Green */
  --primary-foreground: oklch(0.98 0 0);
  --accent: oklch(0.92 0.05 145);
}
```

### Applying Theme in Layout

```typescript
// mrd/apps/healthtalk/app/layout.tsx
import "@mrd/ui/globals.css";
import "@mrd/ui/themes/healthtalk.css";

export default function RootLayout({ children }) {
  return (
    <html>
      <body className="font-sans">{children}</body>
    </html>
  );
}
```

### Runtime Branding via Project.setting

For per-tenant branding overrides, use Medplum Project.setting:

| Setting Key | Description | Example |
|-------------|-------------|---------|
| `brand.id` | Brand identifier | `healthtalk` |
| `brand.name` | Display name | `HealthTalk` |
| `brand.primaryColor` | Primary color (OKLCH) | `oklch(0.55 0.2 250)` |
| `brand.logoUrl` | Logo URL | `https://...` |

```typescript
import { getProjectTheme, generateThemeCSS } from '@mrd/ui/lib/project-theme';

// Server Component
const project = await medplum.getProject();
const theme = getProjectTheme(project, 'healthtalk');
const css = generateThemeCSS(theme);
```

## Agent Overrides

Brands can override base agent configurations:

```yaml
# mrd/apps/healthtalk/.agents/config.yaml
extends: "@mrd/agents"

overrides:
  compliance:
    - nen-7510-healthtalk.mdx   # Brand-specific compliance
  skills:
    - questionnaire-voice.mdx   # HealthTalk-specific skill
```

## Creating a New Brand

1. **Copy template** from existing brand in `mrd/apps/`
2. **Update package.json** with new name (`@mrd/newbrand`)
3. **Create theme** in `mrd/packages/ui/src/themes/newbrand.css`
4. **Customize `.agents/`** with brand-specific MDX overrides
5. **Add specs/** folder with brand specifications
