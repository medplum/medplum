# Multi-Brand Architecture

## Overview

The MEDrecord ecosystem consists of multiple brands, each serving different healthcare use cases while sharing common infrastructure and codebase.

| Brand | Purpose | Primary Users |
|-------|---------|---------------|
| **HealthTalk** | Patient questionnaires, voice/chat intake | Patients, Practitioners |
| **Coachi** | Coaching and mental health support | Coaches, Clients |
| **MedSafe** | Medication safety and compliance | Pharmacists, Patients |
| **MEDrecord** | Core EHR functionality | All healthcare roles |

For tenant (healthcare institution) isolation, see [Multi-Tenant Architecture](./multi-tenant.mdx).

## Brand Theming (Mantine Native)

MEDrecord uses **Mantine theming** - the same system Medplum uses natively. This ensures:

- **Sync-safe** - No Medplum code modifications
- **Single source of truth** - One theme file per brand
- **Native integration** - Works with all Mantine/Medplum components

### Theme File Structure

Each brand has a Mantine theme file:

```
mrd/apps/
├── healthtalk/src/theme.ts    # HealthTalk theme
├── coachi/src/theme.ts        # Coachi theme
├── medsafe/src/theme.ts       # MedSafe theme
└── medrecord/src/theme.ts     # MEDrecord theme
```

### Theme Definition

```typescript
// mrd/apps/healthtalk/src/theme.ts
import { createTheme, MantineColorsTuple } from '@mantine/core';

const healthtalkBlue: MantineColorsTuple = [
  '#E6F0FF',  // 0 - lightest
  '#CCE0FF',  // 1
  '#99C2FF',  // 2
  '#66A3FF',  // 3
  '#3385FF',  // 4
  '#0066CC',  // 5 - primary (default)
  '#0052A3',  // 6
  '#003D7A',  // 7
  '#002952',  // 8
  '#001429',  // 9 - darkest
];

export const healthtalkTheme = createTheme({
  primaryColor: 'healthtalkBlue',
  colors: {
    healthtalkBlue,
  },
  fontFamily: 'Inter, sans-serif',
  headings: {
    fontFamily: 'Inter, sans-serif',
  },
});
```

### Brand Color Palettes

| Brand | Primary Color | Hex | Use Case |
|-------|---------------|-----|----------|
| **HealthTalk** | Blue | `#0066CC` | Trust, healthcare, communication |
| **Coachi** | Green | `#2E7D32` | Growth, wellness, coaching |
| **MedSafe** | Teal | `#00796B` | Safety, medication, reliability |
| **MEDrecord** | Indigo | `#3F51B5` | Professional, enterprise, core |

### Applying Theme

```typescript
// mrd/apps/healthtalk/app/layout.tsx
import { MantineProvider } from '@mantine/core';
import { healthtalkTheme } from '../src/theme';
import '@mantine/core/styles.css';

export default function RootLayout({ children }: { children: React.ReactNode }) {
  return (
    <html lang="nl">
      <body>
        <MantineProvider theme={healthtalkTheme}>
          {children}
        </MantineProvider>
      </body>
    </html>
  );
}
```

## Logo Override

Medplum natively supports logo override via environment variable:

```bash
# .env.local
MEDPLUM_LOGO_URL=https://cdn.healthtalk.ai/logo.svg
```

This works automatically with Medplum's `<Logo />` component - no code changes needed.

For custom logo components in our apps:

```typescript
// mrd/apps/healthtalk/components/brand-logo.tsx
const BRAND_LOGOS = {
  healthtalk: '/logos/healthtalk.svg',
  coachi: '/logos/coachi.svg',
  medsafe: '/logos/medsafe.svg',
  medrecord: '/logos/medrecord.svg',
};

export function BrandLogo({ brand, size = 32 }: { brand: string; size?: number }) {
  return <img src={BRAND_LOGOS[brand]} alt={brand} height={size} />;
}
```

## Agent Overrides

Brands can override base agent configurations:

```yaml
# mrd/apps/healthtalk/.agents/config.yaml
extends: "@mrd/agents"

overrides:
  compliance:
    - nen-7510-healthtalk.mdx
  skills:
    - questionnaire-voice.mdx
```

## Creating a New Brand

1. **Copy template** from existing brand in `mrd/apps/`
2. **Update package.json** with new name (`@mrd/newbrand`)
3. **Create theme.ts** with Mantine `createTheme()` and brand colors
4. **Add logo** to `/public/logos/newbrand.svg`
5. **Customize `.agents/`** with brand-specific MDX overrides
6. **Add specs/** folder with brand specifications
