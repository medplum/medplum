## Per-Organisation SIP Config — Where It Lives

Per-org telephony config is **not** in env vars — env vars are global and cannot vary per customer. Instead:

- Stored in an encrypted key-value store or database table within the gateway
- Managed through a HealthTalk admin UI → Organisation settings → Telephony
- Loaded at runtime by the gateway when a SIP call is triggered
- Rotated independently per org without redeployment

### Config Structure

```typescript
interface OrgTelephonyConfig {
  orgId: string;
  voiceMobileAdapter: 'twilio-mobile';
  voiceSipAdapter: SipAdapterType | null;
  sip?: {
    sbcFqdn: string;
    sbcPort: number;
    authMethod: 'digest' | 'acl';
    authUsername?: string;
    authPassword?: string;  // encrypted at rest
    mediaEncryption: 'required' | 'allowed' | 'disabled';
    callerIdNumber: string;
    elevenLabsPhoneNumberId: string;
  };
  acsConnectionString?: string; // only for teams-calling-plan adapter
}
```
