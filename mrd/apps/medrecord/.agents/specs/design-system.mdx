# Design System

## Overview

MEDrecord brands use **Tailwind CSS** as the UI framework for all custom applications deployed on Vercel/Next.js. This is separate from Medplum's internal Mantine system which remains untouched for upstream compatibility.

## Architecture

### Two Systems, Clearly Separated

| Context | Framework | Location | Owner |
|---------|-----------|----------|-------|
| **Medplum upstream** | Mantine | `/packages/*` | Medplum (do not touch) |
| **MEDrecord brands** | Tailwind | `/mrd/*` | MEDrecord team |

This separation ensures:
- Upstream merges never conflict with our styling
- v0/Vercel/Next.js native Tailwind support
- Brand theming via CSS custom properties

---

## Theming Architecture

### File Structure

```
mrd/packages/ui/src/
├── globals.css              # Base Tailwind tokens (brand-agnostic)
└── themes/
    ├── medrecord.css        # MEDrecord brand colors
    ├── healthtalk.css       # HealthTalk brand colors
    ├── coachi.css           # Coachi brand colors
    └── medsafe.css          # MedSafe brand colors
```

### How It Works

1. **globals.css** - Defines all CSS custom properties with neutral defaults
2. **themes/{brand}.css** - Overrides `--primary`, `--accent`, etc. with brand colors
3. **layout.tsx** - Imports both: `@import globals.css` then `@import themes/{brand}.css`

---

## CSS Custom Properties

All colors use HSL format for simplicity:

```css
:root {
  --primary: 217 65% 39%;           /* HSL values only */
  --primary-foreground: 0 0% 100%;
  --background: 210 20% 98%;
  --foreground: 217 33% 17%;
  /* ... */
}
```

### Token Reference

| Token | Usage |
|-------|-------|
| `--primary` | Buttons, links, active states |
| `--primary-foreground` | Text on primary backgrounds |
| `--secondary` | Secondary buttons, less emphasis |
| `--accent` | Highlights, badges, accents |
| `--muted` | Disabled states, subtle backgrounds |
| `--background` | Page background |
| `--foreground` | Primary text |
| `--border` | Borders, dividers |
| `--ring` | Focus rings |
| `--sidebar` | Sidebar background |

## Typography

All brands use the same font stack:

```css
--font-sans: 'Inter', -apple-system, BlinkMacSystemFont, 'Segoe UI', sans-serif;
--font-mono: 'JetBrains Mono', 'Fira Code', monospace;
```

| Element | Size | Weight |
|---------|------|--------|
| H1 | 2.25rem | 700 |
| H2 | 1.875rem | 600 |
| H3 | 1.5rem | 600 |
| Body | 1rem | 400 |
| Small | 0.875rem | 400 |

## Using in Components

```tsx
// Tailwind classes automatically use CSS custom properties
<button className="bg-primary text-primary-foreground hover:bg-primary/90">
  Submit
</button>

<div className="bg-sidebar text-sidebar-foreground">
  <nav>...</nav>
</div>
```

---

## Dark Mode

Tailwind handles dark mode via the `.dark` class:

```css
.dark {
  --background: 217 33% 10%;
  --foreground: 210 20% 98%;
  /* ... */
}
```

## Shared Components (@mrd/ui)

The `@mrd/ui` package provides shared Tailwind components:

```typescript
import { cn } from '@mrd/ui/lib/utils';
import { Button } from '@mrd/ui/components/button';

// Components use Tailwind + CSS custom properties
<Button variant="primary">Click me</Button>
```
