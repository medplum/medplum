# Environment Variables

All secrets are stored in Vercel Environment Variables. Never commit secrets to the repository.

## HealthTalk App (app.healthtalk.ai)

### MedPlum

| Variable | Description | Example |
|----------|-------------|---------|
| `MEDPLUM_BASE_URL` | MedPlum server URL | `https://medplum.healthtalk.ai` |
| `MEDPLUM_CLIENT_ID` | OAuth client ID | `healthtalk-app` |
| `MEDPLUM_CLIENT_SECRET` | OAuth client secret | `secret_...` |
| `MEDPLUM_PROJECT_ID` | Project identifier | `proj_...` |

## Environment Setup

### Local Development

```bash
# Copy example file
cp .env.example .env.local

# Fill in values from Vercel dashboard or 1Password
```

### Vercel Deployment

1. Go to Vercel Project Settings > Environment Variables
2. Add all required variables for Production, Preview, Development
3. Use Vercel CLI for bulk import: `vercel env pull .env.local`

### Secret Rotation

When rotating secrets:

1. Add new secret with different name (e.g., `TWILIO_AUTH_TOKEN_V2`)
2. Deploy with both old and new
3. Update external services to use new secret
4. Remove old secret
5. Rename new secret to original name
