## Core Design Decisions

1. **Task-first:** Every questionnaire assignment is anchored by a `QuestionnaireProvisioningTask`. Nothing happens without one.

2. **Two questionnaire variants:** With score (PHQ-9, GAD-7 etc.) or without score (intake forms, check-ins). The only structural difference is the presence of `calculatedExpression` items.

3. **MedPlum SDK throughout:** No custom form engine. `<QuestionnaireBuilder />` and `<QuestionnaireForm />` from `@medplum/react` are embedded directly.

4. **Patient Language Layer:** Clinical terminology (LOINC/SNOMED/ICD) stays in the FHIR record. Patients only ever see plain-language text, managed in the builder.

5. **Gateway-first delivery:** All channel integrations live behind a Notification Gateway. HealthTalk never imports Twilio, ElevenLabs, or Postmark SDKs directly.

6. **Six delivery channels:** Web form, WhatsApp interactive, Voice agent (mobile + landline), Email, SMS. All write to the same `QuestionnaireResponse`.

---

## What This Is Not

- This is not a general patient portal.
- This is not a real-time chat system.
- This is not a replacement for the existing session note template system — questionnaires are a second template type within the same Sjablonen module.

## Implementation Phases

| Phase | Components | Dependencies |
|-------|------------|--------------|
| **0. Infrastructure** | Gateway Vercel project, Upstash Redis, env vars | - |
| **1. FHIR Foundation** | Task profile, Questionnaire extensions, MedPlum subscriptions | MedPlum dev project |
| **2. Builder** | Embed `<QuestionnaireBuilder />`, score toggle, LOINC import | LOINC account |
| **3. Web Channel** | Magic link JWT, `<QuestionnaireForm />` wrapper | - |
| **4. Gateway Core** | Notify endpoint, channel selection, status callbacks | Twilio, Postmark |
| **5. WhatsApp** | Interactive flow, session state (Redis), webhook handlers | Twilio WhatsApp |
| **6. Voice** | ElevenLabs agent, Tool calls, mobile + SIP | ElevenLabs BAA |
| **7. Clinician UI** | Task queue, completion view, Teams notifications | Teams webhook |
| **8. Testing** | E2E scenarios, sandbox environments | - |
