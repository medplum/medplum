# Roadmap — Future Integrations

These items are out of scope for the current development sprint but are confirmed direction. They are documented here so the current architecture does not block them.

## Patient Records Screen — Full MedMij ZIB Integration

HealthTalk will build a dedicated patient records screen that surfaces the full set of Dutch MedMij ZIBs (Zorginformatiebouwstenen) as structured clinical data. The questionnaire module is designed to feed directly into this — `QuestionnaireResponse` and `Observation` resources are already stored in MedPlum using nl-core profiles, which is the same foundation the patient records screen will use.

### When the Patient Records Screen Is Built

- Completed questionnaire responses will appear as structured entries in the patient record, not as a separate module
- Scored questionnaires (PHQ-9, GAD-7 etc.) will surface their `Observation` resources alongside other clinical observations in the record
- The ROM trend view (score over time) is a natural component of the records screen rather than a standalone feature
- All ZIBs used will be from `nictiz.fhir.nl.r4.nl-core` — the same package already in use

### ZIBs Relevant to the Questionnaire Module

| ZIB | FHIR R4 Resource | Already in Use |
|-----|------------------|----------------|
| Vragenlijst (Questionnaire) | `Questionnaire` + `QuestionnaireResponse` | Yes |
| MeetUitslag (Observation) | `Observation` | Yes (scores) |
| Contact (Encounter) | `Encounter` | Future (link QR to session) |
| Patient | `Patient` (nl-core-Patient) | Yes |
| Zorgverlener (Practitioner) | `Practitioner` (nl-core-Practitioner) | Yes |
| Behandeling (CarePlan) | `CarePlan` | Future |

**Architectural note:** No changes to the questionnaire module are required for this integration. The FHIR resources are already structured correctly. The patient records screen reads from MedPlum — the data is already there.

## Cal.com — Open Source Calendaring

HealthTalk will adopt **Cal.com** (open source) as its calendaring system for appointment scheduling and attendance tracking. Cal.com replaces or extends the current manual appointment creation in HealthTalk.

### What Cal.com Handles

- Appointment booking (clinician availability, patient self-booking)
- Rescheduling and cancellation
- Attendance confirmation (did the patient show up?)
- Calendar integrations (Google Calendar, Outlook)

### Integration Architecture

```
Cal.com                          MedPlum (FHIR)
   │                                  │
   │  booking.created webhook         │
   └──────────────────────────────────►
                                      │  Create FHIR Appointment resource
                                      │  with questionnaire extension
                                      │
                                      │  Cron fires at T-X
                                      │  → Create Task
                                      │  → Send questionnaire to patient
   │                                  │
   │  booking.attended webhook        │
   └──────────────────────────────────►
                                      │  Update Appointment.status = fulfilled
                                      │  Link to session notes / QR
   │                                  │
   │  booking.cancelled webhook       │
   └──────────────────────────────────►
                                      │  Update Appointment.status = cancelled
                                      │  Cancel pending Task
```

### New Vercel Route Needed

```typescript

//          BOOKING_ATTENDED, BOOKING_NO_SHOW

export async function POST(req: Request) {
  // Verify Cal.com webhook secret
  // Map Cal.com booking → FHIR Appointment
  // Create/update/cancel in MedPlum
  // If BOOKING_CREATED + questionnaires attached → queue Task creation
  // If BOOKING_CANCELLED → cancel pending Tasks
  // If BOOKING_NO_SHOW → update Task businessStatus = 'no-show'
}
```

### Cal.com → FHIR Appointment Mapping

| Cal.com Field | FHIR Appointment Field |
|---------------|------------------------|
| `booking.uid` | `Appointment.identifier` |
| `booking.startTime` | `Appointment.start` |
| `booking.endTime` | `Appointment.end` |
| `booking.attendees[patient]` | `Appointment.participant (Patient)` |
| `booking.organizer` | `Appointment.participant (Practitioner)` |
| `booking.status` | `Appointment.status` |
| `booking.metadata.questionnaires` | HealthTalk extension (questionnaire refs + sendBefore) |

### Attendance Tracking

| Cal.com Event | FHIR Appointment.status | Action |
|---------------|-------------------------|--------|
| `BOOKING_ATTENDED` | `fulfilled` | Session occurred — link QR to Encounter |
| `BOOKING_NO_SHOW` | `noshow` | Patient did not attend — note in dashboard |
| `BOOKING_CANCELLED` | `cancelled` | Cancel pending Task |

### No-Show Handling for Questionnaires

If a patient completes the questionnaire but then does not attend (no-show), the `QuestionnaireResponse` and `Observation` are retained — clinical data is preserved. The Task `businessStatus` is updated to `no-show` for operational tracking.
