# Notification Gateway

The Notification Gateway is a separate Vercel project (`gateway.healthtalk.ai`) that handles all outbound patient communications and provider integrations.

## API Endpoints

### POST /api/notify

Entry point for all notifications. Called by MedPlum subscription.

```typescript
interface NotifyRequest {
  taskId: string;
  action: 'send' | 'remind' | 'cancel';
}

interface NotifyResponse {
  success: boolean;
  channel: string;
  messageId?: string;
  error?: string;
}
```

### POST /api/webhook/twilio

Handles Twilio status callbacks and incoming messages.

```typescript
// Incoming WhatsApp message
interface TwilioWebhook {
  From: string;        // whatsapp:+31612345678
  Body: string;        // User response
  MessageSid: string;
}
```

### POST /api/webhook/elevenlabs

Handles ElevenLabs conversation events.

```typescript
interface ElevenLabsWebhook {
  type: 'call.started' | 'call.ended' | 'tool.call';
  call_id: string;
  data: Record<string, unknown>;
}
```

### POST /api/callback/status

Universal status callback for updating Task status.

```typescript
interface StatusCallback {
  taskId: string;
  status: 'sent' | 'delivered' | 'read' | 'completed' | 'failed';
  channel: string;
  timestamp: string;
  metadata?: Record<string, unknown>;
}
```

## Channel Selection Logic

```typescript
async function selectChannel(task: Task, patient: Patient): Promise<Channel> {
  // 1. Check explicit preference in Task.input
  const preference = getTaskInput(task, 'delivery-channel');
  if (preference) {
    return validateChannel(preference, patient);
  }

  // 2. Check patient communication preferences
  const patientPref = patient.telecom?.find(t => 
    t.extension?.some(e => e.url === 'preferred-channel')
  );
  if (patientPref) {
    return validateChannel(patientPref.value, patient);
  }

  // 3. Default waterfall
  if (hasValidWhatsApp(patient)) return 'whatsapp';
  if (hasValidMobile(patient)) return 'sms';
  if (hasValidEmail(patient)) return 'email';
  
  throw new Error('No valid contact method found');
}
```

## Retry Strategy

| Attempt | Delay | Action |
|---------|-------|--------|
| 1 | Immediate | Send via preferred channel |
| 2 | +1 hour | Retry same channel |
| 3 | +24 hours | Fallback to next channel |
| 4 | +48 hours | Final attempt, alert clinician |
| 5 | - | Mark Task as failed |

## Error Handling


## Security

- All endpoints require `x-medplum-signature` header (webhook verification)
- Patient contact info never logged
- All requests over HTTPS
- Redis data encrypted at rest (Upstash)
