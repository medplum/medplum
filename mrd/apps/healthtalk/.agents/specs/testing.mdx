# Testing Strategy

Sandbox environments and end-to-end test scenarios for the HealthTalk Patient Questionnaire Module.

---

## Sandbox Environments

### Twilio Sandbox

Twilio provides a WhatsApp Sandbox for development. Configure in the Twilio console — no Meta approval needed during development.

```env
TWILIO_WHATSAPP_NUMBER=whatsapp:+14155238886  (sandbox number)
```

### ElevenLabs Conversation Testing

ElevenLabs provides a web-based conversation tester in the agent dashboard. Use this to test the full voice flow before connecting Twilio.

### MedPlum Sandbox

Use a separate MedPlum project for development (`medplumapi-dev.healthtalk.ai`). Never test against production patient data.

### LOINC Import Testing

LOINC API access works the same in dev and production. Use a test questionnaire code (e.g. `55757-9` PHQ-2 — short, 2 items) for initial integration testing.

## End-to-End Test Scenarios

| Scenario | Covers |
|----------|--------|
| Full web form flow | Task creation → magic link → form completion → score → clinician notification |
| WhatsApp interactive — required questions | All question types, required enforcement |
| WhatsApp interactive — optional + enableWhen | Skip logic, conditional questions |
| Voice agent — scored questionnaire | Full PHQ-9 via voice, score calculated |
| Voice agent — quantity items | Weight + height, unit ambiguity |
| Email delivery + magic link | Full flow via email channel |
| SMS delivery + magic link | SMS link to web form |
| Landline call — Teams SIP | Full PHQ-9 via landline, routed through org Teams SBC |
| Answering machine detection | AMD fires, no message left, retry scheduled |
| Teams clinician notification | Completed QR triggers Teams card |
| Teams alert — non-completion | T-0 alert card sent to clinician |
| Appointment cancellation | Task cancelled, patient notified, link deactivated |
| WhatsApp STOP reply | Opt-out handling, Task cancelled |
| Magic link expiry | Expired token → correct message, no data exposed |
| Delivery failure → email fallback | Twilio failure triggers email |
| LOINC import | Code input → fetch → opens in builder |

## Test Data Requirements

### Patients

- Patient with WhatsApp consent
- Patient with email only
- Patient with mobile phone only
- Patient with landline only (SIP)
- Patient with no contact methods (edge case)

### Questionnaires

- PHQ-9 (scored, 9 items)
- PHQ-2 (scored, 2 items — quick test)
- GAD-7 (scored, 7 items)
- Intake form (no score, various question types)
- Questionnaire with enableWhen logic

### Organisations

- Org with Teams Direct Routing
- Org with standalone SBC
- Org without SIP configuration
