# Pre-Development Checklist

All tasks that must be completed before and during development of the HealthTalk Patient Questionnaire Module.

---

## Phase 0: Gateway Project — Must Be Set Up First

- [ ] **Create gateway Vercel project** — `gateway.healthtalk.ai`, separate from main app
- [ ] **Define per-org config store** — decide database or encrypted KV for telephony config (SBC credentials etc.)
- [ ] **Gateway ↔ HealthTalk auth** — generate and exchange `GATEWAY_API_SECRET` and `HEALTHTALK_CALLBACK_SECRET`

---

## Must Be Resolved Before Development Starts

### Legal & Compliance

- [ ] **ElevenLabs Enterprise + BAA** — Legal must sign off before any PHI flows through voice Tool calls

### Third-Party Accounts

- [ ] **Twilio WhatsApp Business** — Apply for and receive approved WhatsApp sender (or confirm sandbox for dev phase)
- [ ] **Meta template approval** — Submit `healthtalk_questionnaire_v1` and `healthtalk_questionnaire_reminder_v1` for Meta approval
- [ ] **LOINC account** — Register organisational account at loinc.org under healthtalk@medrecord.nl
- [ ] **Upstash Redis** — Provision for WhatsApp session state (Vercel integration)
- [ ] **Postmark account** — Provision transactional email service, configure SPF/DKIM for healthtalk.ai
- [ ] **Telnyx SIP account** — Register account, acquire Dutch DID numbers (+31), configure SIP trunk in ElevenLabs for landline channel

### Infrastructure

- [ ] **MedPlum dev project** — Separate project for development, not production
- [ ] **Teams webhook URL** — Configure incoming webhook in the relevant Teams channel for clinician notifications
- [ ] **Teams Phone licence decision** — Decide whether to use Teams Direct Routing (Option 2) in v1 or defer to later phase. If v1: procure Teams Phone licences + SBC.

### Consent

- [ ] **Patient WhatsApp consent** — Confirm whether existing onboarding captures consent for WhatsApp messages
- [ ] **Patient voice call consent** — Confirm whether existing onboarding captures consent for AI outbound calls

### Voice Configuration

- [ ] **ElevenLabs agent Dutch voice** — Select and test Dutch voice model before build
- [ ] **Answering machine detection** — Confirm ElevenLabs / Telnyx AMD is available on the selected plan tier

---

## Must Be Resolved During Development

- [ ] **Clinical validation of patient-language text** — Define review process before any questionnaire is published
- [ ] **Nictiz R4 QuestionnaireProvisioningTask profile URL** — Confirm final URL when Nictiz publishes R4 IG
- [ ] **ROM trend view design** — Score-over-time visualisation for patient history view needs design before frontend build

---

## Nice to Have Before Go-Live

- [ ] **Coachi PGO export** — Forward completed `QuestionnaireResponse` to patient's Coachi PGO
- [ ] **zib2024 migration** — Upgrade from zib2020 nl-core to zib2024 when formally published
- [ ] **SMS fallback** — Basic SMS with magic link for patients with neither WhatsApp nor email

---

## Implementation Phases

| Phase | Components | Dependencies |
|-------|------------|--------------|
| **0. Infrastructure** | Gateway Vercel project, Upstash Redis, env vars | - |
| **1. FHIR Foundation** | Task profile, Questionnaire extensions, MedPlum subscriptions | MedPlum dev project |
| **2. Builder** | Embed `<QuestionnaireBuilder />`, score toggle, LOINC import | LOINC account |
| **3. Web Channel** | Magic link JWT, `<QuestionnaireForm />` wrapper | - |
| **4. Gateway Core** | Notify endpoint, channel selection, status callbacks | Twilio, Postmark |
| **5. WhatsApp** | Interactive flow, session state (Redis), webhook handlers | Twilio WhatsApp |
| **6. Voice** | ElevenLabs agent, Tool calls, mobile + SIP | ElevenLabs BAA |
| **7. Clinician UI** | Task queue, completion view, Teams notifications | Teams webhook |
| **8. Testing** | E2E scenarios, sandbox environments | - |
