# Multi-Tenant Architecture

## Overview

MEDrecord uses a **Gateway-First Multi-Tenant** model where:

- **All authentication flows through Azure B2C via our Gateway**
- **MEDrecord controls all tenant and brand configuration**
- **Medplum serves as the FHIR data layer only**

This ensures total control over identity, tenants, and branding while leveraging Medplum for compliant healthcare data storage.

---

## Core Principles

| Principle | Implementation |
|-----------|----------------|
| **Gateway-First** | All auth and tenant resolution via Gateway, never direct Medplum auth |
| **Azure B2C** | MEDrecord manages all user identities |
| **MEDrecord chooses brand** | We determine which brand (HealthTalk/Coachi/etc) per tenant |
| **Sync-Safe** | All config in `mrd/` namespace or Gateway, never in Medplum upstream code |
| **Total Control** | We control auth, tenants, brands - Medplum is data layer |

---

## The Three Layers

| Layer | Who Decides | Definition | Storage |
|-------|-------------|------------|---------|
| **Brand** | MEDrecord | Our product: HealthTalk, Coachi, MedSafe | Gateway config |
| **Tenant** | MEDrecord | Healthcare institution = Medplum Project | Gateway config |
| **Organization** | Healthcare institution | Internal departments within tenant | Medplum |

## Gateway Response (Single Source of Truth)

Every authenticated request includes tenant context from Gateway:

```typescript
// Gateway auth response - the ONLY source of truth
{
  "auth": {
    "user_id": "azure-b2c-user-id",
    "email": "arts@ziekenhuis.nl",
    "tenant_id": "ziekenhuis-amsterdam",    // Medplum Project ID
    "organization_id": "cardiologie",        // Department (optional)
    "brand": "healthtalk",                   // MEDrecord determines this
    "roles": ["practitioner", "admin"]
  }
}
```

**The app NEVER asks Medplum directly for user identity or tenant. Everything comes from Gateway.**

## Key Points

| Question | Answer |
|----------|--------|
| Who chooses the brand? | **MEDrecord** |
| What does the healthcare institution see? | **Our branding** (HealthTalk/Coachi/etc) |
| Does the institution have their own logo? | **No** - they see our brand |
| What is a tenant? | **Healthcare institution = Medplum Project** |
| What is an organization? | **Department within tenant** |
| Where does auth come from? | **Azure B2C via Gateway** |
| Does Medplum handle auth? | **No** - Gateway handles all auth |

## API Route Pattern (Gateway-First)

```typescript
// app/api/tasks/route.ts
import { verifyGatewayAuth, getMedplumClient } from '@/lib/gateway';

export async function GET(request: Request) {
  // 1. ALWAYS via Gateway - never direct Medplum auth
  const auth = await verifyGatewayAuth(request);
  
  // 2. tenant_id from Gateway (not from request body/params)
  const medplum = getMedplumClient(auth.tenant_id);
  
  // 3. brand from Gateway (for brand-specific logic)
  const brand = auth.brand; // 'healthtalk' | 'coachi' | etc
  
  // 4. Fetch data - automatically scoped to tenant project
  const tasks = await medplum.searchResources('Task', {
    status: 'requested',
  });
  
  return Response.json(tasks);
}
```

## Onboarding a New Tenant

1. **Create Medplum Project** for the tenant
2. **Configure in Gateway** - tenant_id, brand, Medplum credentials
3. **Set up Azure B2C** connection for the institution
4. **Configure Organizations** in Medplum for departments
5. **Set up AccessPolicies** for user roles
6. **Copy template Questionnaires** if needed

All tenant config is in Gateway - the app reads from Gateway at runtime.

## Sync-Safe Architecture

```
Medplum Repository (upstream)
│
├── packages/           <- NEVER TOUCH - sync-safe
│   ├── app/
│   ├── core/
│   ├── react/
│   └── ...
│
└── mrd/                <- OUR NAMESPACE - never overwritten by upstream
    ├── apps/
    │   ├── healthtalk/
    │   ├── coachi/
    │   └── ...
    ├── packages/
    │   └── ui/
    └── docs/
```

| On upstream sync | What happens |
|------------------|--------------|
| `packages/*` changes | We merge, our code untouched |
| `mrd/*` | Never touched by upstream |
| `.gitattributes` | `mrd/** merge=ours` protects our code |
