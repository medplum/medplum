March was a strong month for Medplum. We shipped three patch releases — v5.1.2, v5.1.3, and v5.1.4 — with over 100 commits from 20+ contributors. The biggest themes were **the Provider app (Spaces, messaging, and scheduling)**, **platform and worker infrastructure**, **communications documentation and clinical charting guides**, and **integrations and developer tooling** — all advancing our [2026 roadmap priorities](/blog/2026-roadmap).

## Features

### Provider App: Spaces, Messaging, and Scheduling

<img src="https://github.com/techdavidy.png" alt="David Yanez" width="50" height="50" style={{borderRadius: '50%'}} /> **[David Yanez](https://github.com/techdavidy)**

The [Provider app](https://provider.medplum.com) continued to mature across [Spaces](https://provider.medplum.com/Spaces/Communication), messaging, and calendar workflows, supporting the [Provider Application roadmap goal](/docs/provider):

- **Spaces system prompts on `Communication` resources** — Spaces configuration now stores system prompts as `Communication` resources for clearer modeling and reuse
- **Component preview, code, and FHIR resources in Spaces** — Richer previews and visibility into generated UI components, code, and underlying resources inside the Spaces chat experience
- **Translator tool looping** — The translator tool can iterate until the user’s request is satisfied, improving reliability for multi-step AI workflows
- **Updated default language model** — Spaces adopts newer model defaults for improved quality and consistency
- **Messaging and Communications** — A dedicated **payload** tab on Communications, **eFax** integration, and ThreadChat improvements including subject on communications and clearer file metadata for uploads and search
- **Scheduling** — Switch between practitioner schedules from the calendar, more robust **Find** pane behavior when service types omit a system, **visit** setup immediately after [`$book`](/docs/scheduling/appointment-book), and schedule pickers that resolve actors via `Schedule:actor`
- **Revenue cycle (Candid)** — Claim status visibility in the Provider app and use of external identifiers when a Candid encounter id is absent
- **SMART App Launch** — Support for `fhirContext` on Smart App Launch, and the Provider app respects an existing `login` query parameter when present
- **Spaces bots in demo bots** — Demo bots for Spaces are easier to discover and use in development environments

Standardized tab navigation across the Provider app by [Kevin Wadeshaw](https://github.com/kevinwadeshaw) improves consistency across high-traffic screens.

### Scheduling: `$find`, `$book`, and Demos

<img src="https://github.com/noahsilas.png" alt="Noah Silas" width="50" height="50" style={{borderRadius: '50%'}} /> **[Noah Silas](https://github.com/noahsilas)**

Scheduling advances the [Scheduling roadmap initiative](/docs/scheduling) and self-scheduling direction:

- **`$book` populates `Appointment.slot`** — Booked appointments now carry slot references populated by the server (contributed by Dmytro Smyk)
- **Improved `$find` service type matching** — More accurate matching when resolving available slots
- **Removal of generic (“wildcard”) availability** — Clarifies availability semantics and reduces ambiguous slot definitions
- **[FooMedical scheduling demo](https://github.com/medplum/medplum/tree/main/examples/foomedical)** — The patient-facing demo showcases updated scheduling flows

Demo schedule content and calendar examples were refreshed by [Reshma Khilnani](https://github.com/reshmakh).

### Enterprise Scale: Workers, Auth, and Data Path

<img src="https://github.com/codyebberson.png" alt="Cody Ebberson" width="50" height="50" style={{borderRadius: '50%'}} /> **[Cody Ebberson](https://github.com/codyebberson)** and <img src="https://github.com/mattlong.png" alt="Matt Long" width="50" height="50" style={{borderRadius: '50%'}} /> **[Matt Long](https://github.com/mattlong)**

Platform work advances [Enterprise Scale & Infrastructure](/blog/2026-roadmap) themes:

- **Dispatch and background workers** — A dedicated dispatch worker path, configurable background workers, CDK support for worker-only services, and conditional deployment of worker infrastructure
- **`auth/me` includes project features** — Clients can read enabled project features from the authenticated session for cleaner feature gating
- **Marketplace definition resources** — Foundation resources for marketplace packages and related automation metadata ([package resource](/docs/api/fhir/medplum/package))
- **New Bot `$init` operation** — Streamlines initializing Bot content and related resources for repeatable deployments ([Bots overview](/docs/bots))
- **Extracted authentication React components** — Shared building blocks for sign-in and session flows across apps

<img src="https://github.com/mattwiller.png" alt="Matt Willer" width="50" height="50" style={{borderRadius: '50%'}} /> **[Matt Willer](https://github.com/mattwiller)**

- **Presigned URLs for Binary upload** — Efficient, secure client uploads aligned with common storage patterns
- **Rate limiting and polling** — In-memory caching for rate-limited IPs, faster login flows, AsyncJob polling via the system repository, and PostgreSQL transaction idle timeouts tuned for busy deployments
- **Metrics and consistency** — Broader FHIR interaction metrics from the system repository and transaction consistency fixes in database reads

<img src="https://github.com/ThatOneBro.png" alt="Derrick Farris" width="50" height="50" style={{borderRadius: '50%'}} /> **[Derrick Farris](https://github.com/ThatOneBro)**

Further hardening for [WebSocket subscriptions](/docs/subscriptions): subscription token lifecycle, stale subscription detection against user limits, FHIRcast topic key refresh, and HL7 client behavior that **warns** instead of failing on unknown message control IDs.

<img src="https://github.com/techdavidy.png" alt="David Yanez" width="50" height="50" style={{borderRadius: '50%'}} /> **[David Yanez](https://github.com/techdavidy)** and <img src="https://github.com/noahsilas.png" alt="Noah Silas" width="50" height="50" style={{borderRadius: '50%'}} /> **[Noah Silas](https://github.com/noahsilas)**

- **`useSearch` typing** — Search results use `WithId<T>` for safer access to server-assigned ids
- **Default SMART launch behavior** — SMART launches default to opening in a new tab ([Maddy Li](https://github.com/maddyli))
- **Reference display fallback** — Graceful UUID fallback when a reference lacks a display string ([Maddy Li](https://github.com/maddyli))

Additional React improvements include optional formatting hooks for display components by [nate-watkins](https://github.com/nate-watkins) and configurable count behavior in `ValueSetAutocomplete` by [Aditya Suri](https://github.com/adityasuri).

**Medplum client options** — Configurable maximum retry time and default `maxRetries` support more resilient integrations ([Cody Ebberson](https://github.com/codyebberson), [Aditya Suri](https://github.com/adityasuri)).

**Clinical and data quality**

- Fixed `mapReferenceRange` so numeric reference ranges map cleanly into FHIR `Range` for observations (contributed by [Amanda McGivern](https://github.com/amcgivern))

**Server and storage**

- Return HTTP 400 when PostgreSQL rejects date/time values that overflow field bounds, instead of surfacing a generic database error (contributed by Léandre Chamberland-Dozois)
