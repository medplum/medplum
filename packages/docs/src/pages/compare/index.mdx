# Compare Medplum

Healthcare software is a broad category that includes hospital EHRs, ambulatory EHRs, interoperability platforms, backend developer tools, and healthcare infrastructure products.

Medplum is not a traditional hospital EHR. Medplum is a developer platform for building healthcare applications and clinical systems using open standards such as FHIR, OAuth2, and SMART on FHIR.

Organizations evaluating Medplum often compare it to a wide range of products depending on their technical requirements, care setting, and stage of growth.

This guide provides an overview of common comparison points and helps explain where different products fit in the healthcare technology ecosystem.

## Healthcare Infrastructure Platforms

These platforms are closest to Medplum architecturally and are commonly evaluated by engineering teams building healthcare applications, interoperability systems, and clinical infrastructure.

- Medplum vs Microsoft FHIR Server for Azure
- [Medplum vs HAPI FHIR](/compare/hapi)
- Medplum vs Firely Server
- [Medplum vs Aidbox](/compare/aidbox)
- [Medplum vs OystEHR](/compare/oystehr)

Typical considerations:

- FHIR API support
- extensibility
- hosted vs self-hosted deployment
- operational complexity
- authentication and authorization
- developer experience
- scalability
- healthcare integrations

## Identity & Access Management

Modern healthcare applications often require OAuth2, OpenID Connect, SMART on FHIR, multi-tenant authorization, and fine-grained access control.

Some organizations compare Medplum to identity infrastructure products because Medplum includes integrated authentication and authorization capabilities for healthcare applications.

- Medplum vs Keycloak
- Medplum vs Ory

Typical considerations:

- OAuth2 and OIDC support
- SMART on FHIR compatibility
- user and project management
- RBAC and ABAC authorization
- multi-tenant architecture
- healthcare-specific identity workflows

## Backend Developer Platforms

Many teams evaluating Medplum come from modern web and mobile development backgrounds and are familiar with backend developer platforms such as Firebase or Supabase.

Medplum is often used similarly — as application infrastructure — but with healthcare-specific APIs, data models, compliance features, and interoperability capabilities.

- Medplum vs Firebase
- Medplum vs Supabase
- Medplum vs AWS Amplify

Typical considerations:

- developer onboarding
- realtime APIs
- authentication
- file storage
- serverless workflows
- healthcare compliance
- FHIR-native data models
- healthcare interoperability

## Choosing the Right Platform

Different healthcare platforms are optimized for different goals.

Some organizations need a hospital-scale operational EHR. Others need a programmable healthcare backend, a patient engagement platform, or a modern interoperability layer.

Medplum is designed for teams building:

- digital health applications
- healthcare APIs
- provider and patient portals
- clinical workflows
- interoperability systems
- operational healthcare infrastructure
- healthcare AI applications
- custom care delivery platforms

If you're evaluating healthcare technology platforms and are unsure where to start, these comparison guides can help clarify the tradeoffs, capabilities, and intended use cases of each system.
