<div> <a name="bnr"></a> <p>Note that the <Link to="/docs/api/fhir/resources/supplyrequest">SupplyRequest</Link> records the fact that a request was made. To actually act on that request, additional workflow beyond simply the existence of a <Link to="/docs/api/fhir/resources/supplyrequest">SupplyRequest</Link> is required. This can be achieved by using an <a href="/docs/api/fhir/resources/task">Task</a> resource, with the <Link to="/docs/api/fhir/resources/supplyrequest">SupplyRequest</Link> referenced from the <Link to="/docs/api/fhir/resources/task">Task</Link>.focus, or by using the <Link to="/docs/api/fhir/resources/supplyrequest">SupplyRequest</Link> resource in the context of an messaging or service workflow where the request is explicit or implicit.  The <a href="/docs/api/fhir/resources/supplydelivery">SupplyDelivery</a> resource represents the fulfillment as a result of <Link to="/docs/api/fhir/resources/supplyrequest">SupplyRequest</Link> being acted upon. </p> <p>The <Link to="/docs/api/fhir/resources/supplyrequest">SupplyRequest</Link> resource is used for <em>inventory management</em>.       When requesting medication, substances and devices when there is a patient focus or instructions regarding their use, <a href="/docs/api/fhir/resources/devicerequest">DeviceRequest</a> or <a href="/docs/api/fhir/resources/medicationrequest">MedicationRequest</a> should be used instead </p> </div>
