This can be achieved by using an <a href="/docs/api/fhir/resources/task">Task</a> resource, with the <a href="/docs/api/fhir/resources/supplyrequest">SupplyRequest</a> referenced from the <a href="/docs/api/fhir/resources/task">Task</a>.focus, or by using the <a href="/docs/api/fhir/resources/supplyrequest">SupplyRequest</a> resource in the context of an messaging or service workflow where the request is explicit or implicit.  The <a href="/docs/api/fhir/resources/supplydelivery">SupplyDelivery</a> resource represents the fulfillment as a result of <a href="/docs/api/fhir/resources/supplyrequest">SupplyRequest</a> being acted upon.   </p>   <p>The <a href="/docs/api/fhir/resources/supplyrequest">SupplyRequest</a> resource is used for <em>inventory management</em>.       When requesting medication, substances and devices when there is a patient focus or instructions regarding their use, <a href="/docs/api/fhir/resources/devicerequest">DeviceRequest</a> or <a href="/docs/api/fhir/resources/medicationrequest">MedicationRequest</a> should be used instead   </p>  </div>
