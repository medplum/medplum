# PractitionerRole

A specific set of Roles/Locations/specialties/services that a practitioner may perform at an organization for a period of time.

  <TabItem value="usage" label="Usage">
    <div> <a name="scope"></a>  <p> <a href="/docs/api/fhir/resources/practitionerrole">PractitionerRole</a> covers the recording of the location and types of services that <a href="/docs/api/fhir/resources/practitioner">Practitioners</a> are able to provide for an organization. </p> <p> The role, specialty, <a href="/docs/api/fhir/resources/location">Location</a> telecom and <a href="/docs/api/fhir/resources/healthcareservice">HealthcareService</a> properties can be repeated if required in other instances of the <a href="/docs/api/fhir/resources/practitionerrole">PractitionerRole</a>. Some systems record a collection of service values for a single location, others record the  single service and the list of locations it is available. Both are acceptable options for representing this data.<br/> Where availability, telecom, or other details are not the same across all healthcareservices, or locations  a seperate <a href="/docs/api/fhir/resources/practitionerrole">PractitionerRole</a> instance should be created. </p> </div>
    </TabItem>
  <TabItem value="relationships" label="Relationships">
    <div> <a name="bnr"></a>  <p>  Qualifications (from the <a href="/docs/api/fhir/resources/practitioner">Practitioner</a> resource) do not imply a Role, but might be considered when an <a href="/docs/api/fhir/resources/organization">Organization</a>    allocates practitioners to roles within their organization, and could provide useful information (such as expiry information)   which could need to be tracked in some situations to ensure they continue to be eligible for a specific role. </p> <p>   The <a href="/docs/api/fhir/resources/careteam">CareTeam</a> resource is also often used to provide details of a role that a practitioner is   allocated to perform, but is usually scoped to a much finer granularity of care, and often within the specific context   of a <a href="/docs/api/fhir/resources/patient">Patient</a>, or functional role (e.g. Crisis planning team). In contrast the <a href="/docs/api/fhir/resources/practitionerrole">PractitionerRole</a>   is used in a more general sense to cover all the places that the practitioner is allocated to work (and specific details   relevant to that role - such as a specific contact number, or electronic services endpoint). </p> </div>
    </TabItem>
  <TabItem value="backgroundAndContext" label="Background and Context">
    <div> <a name="bnc"></a>  <p> <a href="/docs/api/fhir/resources/practitioner">Practitioner</a> performs different roles within the same or even different organizations. Depending on jurisdiction and custom,  it may be necessary to maintain a specific <a href="/docs/api/fhir/resources/practitioner">Practitioner</a> Resource for each such role or have a single <a href="/docs/api/fhir/resources/practitioner">Practitioner</a> with multiple roles. The role can be limited to a specific period, after which authorization for this role ends. Note that the represented organization need not necessarily be the (direct) employer of a <a href="/docs/api/fhir/resources/practitioner">Practitioner</a>. </p> </div>
    </TabItem>
