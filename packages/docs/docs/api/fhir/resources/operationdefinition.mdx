  <TabItem value="usage" label="Usage">
    <div> <a name="scope"></a>  <p> The <a href="/docs/api/fhir/resources/operationdefinition">OperationDefinition</a> resource provides a formal computable definition of an <a href="https://www.hl7.org/fhir/operations.html">operation</a> or <a href="https://www.hl7.org/fhir/search.html#advanced">a named query</a>. The <a href="/docs/api/fhir/resources/operationdefinition">OperationDefinition</a> serves two principal purposes: </p> <ul>  <li>To allow for automatic determination of system compatibility</li>  <li>To allow for dynamic generation of forms to drive the operations</li> </ul> <p> See below for further information about these, and about how operations and named queries are executed. </p> </div>
    </TabItem>
  <TabItem value="relationships" label="Relationships">
    <div> <a name="bnr"></a>  <p> <a href="/docs/api/fhir/resources/operationdefinition">OperationDefinitions</a> are published to define operations that servers can implement in a common fashion.  The FHIR specification itself describes some (see below), and other organizations (including  IHE, national programs, jurisdictions and vendors) are able to publish additional <a href="/docs/api/fhir/resources/operationdefinition">OperationDefinitions</a>. </p> <p> <a href="/docs/api/fhir/resources/operationdefinition">OperationDefinition</a> resources are referred to from two different places: </p> <ul>  <li>From a <a href="/docs/api/fhir/resources/capabilitystatement">CapabilityStatement</a>, to declare what operations a system does or should implement</li>  <li>From another <a href="/docs/api/fhir/resources/operationdefinition">OperationDefinition</a> resource. This allows for a server to describe a limited implementation of a standard operation, or to allow traceability if the server has to rename the operation due to a name clash</li> </ul> </div>
