# Organization

A formally or informally recognized grouping of people or organizations formed for the purpose of achieving some form of collective action. Includes companies, institutions, corporations, departments, community groups, healthcare practice groups, payer/insurer, etc.

  <TabItem value="relationships" label="Relationships">
    <div> <a name="bnr"></a>  <p> The <a href="/docs/api/fhir/resources/organization">Organization</a> resource is used for collections of people that have come together to achieve an objective.   The <a href="/docs/api/fhir/resources/group">Group</a> resource is used to identify a collection of people (or animals, devices, etc.)  that are gathered for the purpose of analysis or acting upon, but are not expected to act themselves. </p>  <p>   The <a href="/docs/api/fhir/resources/organization">Organization</a> resource often exists as a hierarchy of organization resources, using the <i>part-of</i> property to provide   the association of the child to its parent organization.<br/>   This organizational hierarchy helps communicate the conceptual structure, whereas the <a href="/docs/api/fhir/resources/location">Location</a> resource provides the physical   representation of the hierarchy.<br/>   The linkage between <a href="/docs/api/fhir/resources/organization">Organization</a> and <a href="/docs/api/fhir/resources/location">Location</a> is from each point in the location hierarchy to the appropriate level in the    <a href="/docs/api/fhir/resources/organization">Organization</a> hierarchy. These links don't all have to be to the top level <a href="/docs/api/fhir/resources/organization">Organization</a>.<br/>   When populating the organization and location hierarchies there is often not a clear distinction between these 2, however   to assist in making the decision, <a href="/docs/api/fhir/resources/location">Locations</a> are always used for recording where a service occurs, and hence where encounters   and observations are associated. The <a href="/docs/api/fhir/resources/organization">Organization</a> property on these resources might not be the location where the service took place.  </p> </div>
