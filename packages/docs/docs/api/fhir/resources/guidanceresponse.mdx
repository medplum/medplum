# GuidanceResponse

A guidance response is the formal response to a guidance request, including any output parameters returned by the evaluation, as well as the description of any proposed actions to be taken.

<div> <a name="bnc"></a> <p>The <Link to="/docs/api/fhir/resources/guidanceresponse">GuidanceResponse</Link> resource is used to represent the result of invoking a decision support service. It provides a container for the status of the response, any warnings or messages returned by the service, as well as any output data from the service and any suggested actions to be performed.</p> <p>Use cases for the <Link to="/docs/api/fhir/resources/guidanceresponse">GuidanceResponse</Link> resource include:</p> <ul> <li>Keeping a record of past service requests, including the ability to reference the context of the request such as the disease state</li> <li>Representing additional data requirements for subsequent service requests</li> <li>Representing Public Health case reporting and recording</li> <li>Sharing the results of decision support guidance and determination of reportability with clinicians and public health agencies</li> </ul> <p>In addition, the <Link to="/docs/api/fhir/resources/guidanceresponse">GuidanceResponse</Link> resource is aligned with the CDS Hooks response structure and so provides a mechanism to record the results of CDS Hooks interactions. For a detailed discussion of how the <Link to="/docs/api/fhir/resources/guidanceresponse">GuidanceResponse</Link> resource can be used for this purpose, refer to the <a href="https://www.hl7.org/fhir/clinicalreasoning-cds-on-fhir.html#evaluation">Evaluation</a> topic in the Clinical Reasoning module.</p> </div>
