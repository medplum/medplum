  <TabItem value="usage" label="Usage">
<div> <a name="scope"></a> <p> Operation outcomes are sets of error, warning and information messages that provide detailed information about the outcome of an attempted system operation.   They are provided as a direct system response, or component of one, and provide information about the outcome of the operation. </p> <p> The <Link to="/docs/api/fhir/resources/operationoutcome">OperationOutcome</Link> resource is used in the following circumstances: </p> <ul>  <li>When a <a href="https://www.hl7.org/fhir/http.html#operations">RESTful interaction</a> or <a href="https://www.hl7.org/fhir/operations.html">operation</a> fails</li>  <li>As the response on a <a href="https://www.hl7.org/fhir/http.html#validate">validation operation</a> to provide information about the outcome</li>  <li>As part of a message response, usually when the message has not been processed correctly</li>  <li>As the response to a batch/transaction, when requested</li>  <li>As part of a search's <Link to="/docs/api/fhir/resources/bundle">Bundle</Link> response containing information about the search</li> </ul> </div>
  </TabItem>
  <TabItem value="relationships" label="Relationships">
<div> <a name="bnr"></a> <p> This resource is not used for reporting clinical or workflow issues associated with a proposed or ongoing action;  these would be handled using <a href="/docs/api/fhir/resources/detectedissue">DetectedIssue</a> or other resources. <Link to="/docs/api/fhir/resources/operationoutcome">OperationOutcome</Link>   is not designed to be persisted or referenced from other parts of the workflow.  </p> <p> It is possible to have both <code>OperationOutcome</code> and <a href="/docs/api/fhir/resources/detectedissue">DetectedIssue</a> together, where the <Link to="/docs/api/fhir/resources/operationoutcome">OperationOutcome</Link> might  indicate that a requested action was rejected due to a clinical issue and the <Link to="/docs/api/fhir/resources/detectedissue">DetectedIssue</Link> provides the details of the issue. </p> </div>
