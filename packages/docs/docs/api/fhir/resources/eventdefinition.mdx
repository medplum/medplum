# EventDefinition

The <Link to="/docs/api/fhir/resources/eventdefinition">EventDefinition</Link> resource provides a reusable description of when a particular event can occur.

<div> <a name="bnc"></a> <p><strong>This resource is a <em>definition</em> resource from a FHIR workflow perspective</strong> - see <a href="https://www.hl7.org/fhir/workflow.html">Workflow</a>, specifically <a href="https://www.hl7.org/fhir/workflow.html#definition">Definition</a>.</p> <p>The <Link to="/docs/api/fhir/resources/eventdefinition">EventDefinition</Link> provides a reusable description of an event. The resource supports describing different kinds of events, including named events, periodic events, and data-based events. For each of these, the resource also supports a formal description of the event. For example, a 'monitor-emergency-admissions' event can be a named event, but also provide a formal description of the event as monitoring for encounters that occur in emergency department locations.</p> </div>

  <TabItem value="relationships" label="Relationships">
<div> <p>The <Link to="/docs/api/fhir/resources/eventdefinition">EventDefinition</Link> resource is intended to be used to provide a reusable description of events for use by other resources that require event definition, such as <Link to="/docs/api/fhir/resources/subscription">Subscription</Link>. <Link to="/docs/api/fhir/resources/eventdefinition">EventDefinition</Link> is a definition of an event that could occur, whereas <Link to="/docs/api/fhir/resources/subscription">Subscription</Link> is asking for notification of events when they occur.</p> </div>
