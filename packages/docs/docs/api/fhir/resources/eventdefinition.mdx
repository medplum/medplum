# EventDefinition

The <a href="/docs/api/fhir/resources/eventdefinition">EventDefinition</a> resource provides a reusable description of when a particular event can occur.

  <Tabs queryString="intro-section">
  <TabItem value="usage" label="Usage" default>
    <div> <a name="bnc"></a>  <p><strong>This resource is a <em>definition</em> resource from a FHIR workflow perspective</strong> - see <a href="https://www.hl7.org/fhir/workflow.html">Workflow</a>, specifically <a href="https://www.hl7.org/fhir/workflow.html#definition">Definition</a>.</p>  <p>The <a href="/docs/api/fhir/resources/eventdefinition">EventDefinition</a> provides a reusable description of an event. The resource supports describing different kinds of events, including named events, periodic events, and data-based events. For each of these, the resource also supports a formal description of the event. For example, a 'monitor-emergency-admissions' event can be a named event, but also provide a formal description of the event as monitoring for encounters that occur in emergency department locations.</p> </div>
    </TabItem>
  <TabItem value="relationships" label="Relationships">
    <div>  <p>The <a href="/docs/api/fhir/resources/eventdefinition">EventDefinition</a> resource is intended to be used to provide a reusable description of events for use by other resources that require event definition, such as <a href="/docs/api/fhir/resources/subscription">Subscription</a>. <a href="/docs/api/fhir/resources/eventdefinition">EventDefinition</a> is a definition of an event that could occur, whereas <a href="/docs/api/fhir/resources/subscription">Subscription</a> is asking for notification of events when they occur.</p> </div>
    </TabItem>
  
  
</Tabs>
