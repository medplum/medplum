# Endpoint

The technical details of an endpoint that can be used for electronic services, such as for web services providing XDS.b or a REST endpoint for another FHIR server. This may include any security context information.

<div> <a name="scope"></a>          <p>         An endpoint describes the technical details of a location that can be connected to for the delivery/retrieval of information.         Sufficient information is required to ensure that a connection can be made securely, and appropriate data transmitted as         defined by the endpoint owner. This is not a description of details of the current system, as found in <a href="/docs/api/fhir/resources/capabilitystatement">CapabilityStatement</a>, but of         another (potentially external) system.<br/>         These may be locally hosted services, regional services, or national service.     </p>     <p>       These resources are typically used to identify where to locate endpoint details for:     </p>     <ul>       <li>Questionnaires: Where to send information (currently an SDC extension with just the address)</li>       <li>ValueSet: Where related Terminology Services can be found (where not local)</li>       <li>Subscription: The destination to send the subscribed data (or to pull)</li>       <li>Messaging: (currently defined in the Message Header, but only as the address)</li>       <li>Referrals: Where to send referral requests<br/>           (linked to the services directory resources - <Link to="/docs/api/fhir/resources/organization">Organization</Link>/Location/Practitioner/HealthcareService)</li>       <li>Referrals - Templates: Where to locate referral templates (Questionnaires)<br/>           (linked to the services directory resources - <Link to="/docs/api/fhir/resources/organization">Organization</Link>/Location/Practitioner/HealthcareService)</li>       <li>CarePlans: Where a shared <Link to="/docs/api/fhir/resources/careplan">CarePlan</Link> can be found</li>       <li>Scheduling: Where to lookup to discover schedules/availability information<br/>           (linked to the services directory resources - <Link to="/docs/api/fhir/resources/organization">Organization</Link>/Location/Practitioner/HealthcareService)</li>       <li>Scheduling: Where to lookup to send appointment requests<br/>           (linked to the services directory resources - <Link to="/docs/api/fhir/resources/organization">Organization</Link>/Location/Practitioner/HealthcareService)</li>       <li>Patient/Person: <Link to="/docs/api/fhir/resources/location">Location</Link> of Master <Link to="/docs/api/fhir/resources/patient">Patient</Link>/Person Indexes</li>       <li>Service Provider Directories: <Link to="/docs/api/fhir/resources/location">Location</Link> of related directories (parent/child/federated) </li>   <li>DICOM/imaging: <Link to="/docs/api/fhir/resources/location">Location</Link> of where to query for imaging content and metadata (QIDO-RS, WADO-RS, or WADO-URI)</li>     </ul>   </div>

<div>      <a name="bnr"></a>  <h3 class="self-link-parent"> <Link to="/docs/api/fhir/resources/capabilitystatement">CapabilityStatement</Link> <a href="/docs/api/fhir/resources/endpoint" title="link to here" class="self-link"></a></h3>     <p>       The endpoint is distinct from a capability statement in that the <Link to="/docs/api/fhir/resources/capabilitystatement">CapabilityStatement</Link> statement describes the entire capability       of a server (and in the metadata case, just this server)<br/>       Where the endpoint resource describes the technical details for how to connect, and for what purposes        (which could be a small sub-set of the server's capabilities, and might not be a FHIR endpoint).     </p> <h3 class="self-link-parent"> <Link to="/docs/api/fhir/datatypes/contactpoint">ContactPoint</Link> <a href="https://www.hl7.org/fhir/endpoint.html#8.9.2.2" title="link to here" class="self-link"></a></h3> <p> <Link to="/docs/api/fhir/resources/endpoint">Endpoints</Link> are used to facilitate system to system communication, providing the details of the addressing of the system, purpose of use,    protocols required/provided, and any other details required to communicate between the systems. (such as configuration parameters,    require headers ...) The address value in the endpoint can only be used in the context of the provided details. </p><p> E.g. If the endpoint has a URL for an XDS service, using that in a browser by a user is meaningless. </p><p> E.g.2 If the endpoint has a US Direct address, using this to send a normal email message will not work, as they are not true email addresses (even though they appear to be) </p><p> A <Link to="/docs/api/fhir/datatypes/contactpoint">ContactPoint</Link> contains contact information, such as a phone number, web address that a user can directly interact with, no additional  information would be required to use the value. The value is generally used to communicate with an entity directly, not a system. </p>   </div>
