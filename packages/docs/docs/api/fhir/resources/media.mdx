# Media

A photo, video, or audio recording acquired or used in healthcare. The actual content may be inline or provided by direct reference.

    <div> <a name="bnr"></a>   <p> This resource captures a specific type of <a href="/docs/api/fhir/resources/observation">Observation</a> - an <a href="/docs/api/fhir/resources/observation">Observation</a> whose value is audio, video or image data.   This resource is the preferred representation of such forms of information as it exposes the metadata relevant for  interpreting the information.  However, in some legacy environments, media information may occasionally appear in  <a href="/docs/api/fhir/resources/observation">Observation</a> instead.  Systems should be aware of this possibility. </p> <p> The <a href="/docs/api/fhir/resources/media">Media</a> resource is able to contain medical images in a DICOM format. These images may also be made accessible  through an <a href="/docs/api/fhir/resources/imagingstudy">ImagingStudy</a> resource, which provides a direct reference to the image to a <a href="https://www.hl7.org/fhir/ftp://medical.nema.org/medical/dicom/final/sup161_ft.pdf">WADO-RS server</a>. </p> <p> For such images, the WADO-RS framework is a preferred method for representing the images - the WADO-RS service may include rendering the image with annotations and display parameters from an associated DICOM  presentation state, for instance.  </p> <p> On the other hand, the media resource allows for a robust transfer of an image across boundaries where the WADO-RS service is not available. For this reason, medical images can also be represented in  a <a href="/docs/api/fhir/resources/media">Media</a> resource, but the <a href="/docs/api/fhir/resources/media">Media</a>.content.url should provide a reference to a source WADO-RS service  for the image. </p> </div>
