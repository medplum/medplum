# Observation

Measurements and simple assertions made about a patient, device or other subject. US Core Profiles related to Observations are as follows:

- [US Core Laboratory Result profile](https://hl7.org/fhir/us/core/stu3.1.1/StructureDefinition-us-core-observation-lab.html)
- [US Core Smoking Status Observation profile](https://hl7.org/fhir/us/core/stu3.1.1/StructureDefinition-us-core-smokingstatus.html).
- [US Core Pulse Oximetry Profile](https://hl7.org/fhir/us/core/stu3.1.1/StructureDefinition-us-core-pulse-oximetry.html)
- [US Core Pediatric BMI for Age Profile](https://hl7.org/fhir/us/core/stu3.1.1/StructureDefinition-pediatric-bmi-for-age.html)
- [US Core Pediatric Weight for Height](https://hl7.org/fhir/us/core/stu3.1.1/StructureDefinition-pediatric-weight-for-height.html)
- [US Core Pediatric Head Circumference](https://hl7.org/fhir/us/core/stu3.1.1/StructureDefinition-head-occipital-frontal-circumference-percentile.html)
- [US Core Observation Survey](http://hl7.org/fhir/us/core/StructureDefinition-us-core-observation-survey.html)
- [US Core Observation Social History](http://hl7.org/fhir/us/core/StructureDefinition-us-core-observation-social-history.html)
- [US Core Clinical Test Result](http://hl7.org/fhir/us/core/StructureDefinition-us-core-observation-clinical-test.html)
- [US Core Observation Sexual Orientation](http://hl7.org/fhir/us/core/StructureDefinition-us-core-observation-sexual-orientation.html)

<div>          <p>This resource is an <a href="https://www.hl7.org/fhir/workflow.html#event">         <em>event resource</em>       </a> from a FHIR workflow perspective - see <a href="https://www.hl7.org/fhir/workflow.html">Workflow</a>.</p>     <p>Observations are a central element in healthcare, used to support diagnosis, monitor progress, determine baselines and patterns and even capture demographic characteristics.  Most observations are simple name/value pair assertions with some metadata, but some observations group other observations together logically, or even are multi-component observations. Note that the  <a href="/docs/api/fhir/resources/diagnosticreport">DiagnosticReport</a> resource provides a clinical or workflow context for a set of observations and the <Link to="/docs/api/fhir/resources/observation">Observation</Link> resource is referenced by <Link to="/docs/api/fhir/resources/diagnosticreport">DiagnosticReport</Link> to represent laboratory, imaging, and other clinical and diagnostic data to form a complete report. </p>     <p>Uses for the <Link to="/docs/api/fhir/resources/observation">Observation</Link> resource include:</p>     <ul>       <li>Vital signs such as <a href="https://www.hl7.org/fhir/observation-example.html">body weight</a>,  <a href="https://www.hl7.org/fhir/observation-example-bloodpressure.html">blood pressure</a>, and <a href="https://www.hl7.org/fhir/observation-example-f202-temperature.html">temperature</a>       </li>       <li>Laboratory Data like <a href="https://www.hl7.org/fhir/observation-example-f001-glucose.html">blood glucose</a>, or an <a href="https://www.hl7.org/fhir/observation-example-f205-egfr.html">estimated GFR</a>       </li>       <li>Imaging results like  <a href="https://www.hl7.org/fhir/observation-example-bmd.html">bone density</a> or fetal measurements</li>       <li>Clinical Findings<sup>†</sup> such as  <a href="https://www.hl7.org/fhir/observation-example-abdo-tender.html">abdominal tenderness</a>       </li>       <li>Device measurements such as  <a href="https://www.hl7.org/fhir/observation-example-sample-data.html">EKG data</a> or  <a href="https://www.hl7.org/fhir/observation-example-satO2.html">Pulse Oximetry data</a>       </li>       <li>Clinical assessment tools such as <a href="https://www.hl7.org/fhir/observation-example-5minute-apgar-score.html">APGAR</a> or a <a href="https://www.hl7.org/fhir/observation-example-eye-color.html">eye-color</a>       </li>       <li>Social history like tobacco use,  family support, or cognitive status</li>       <li>Core characteristics like pregnancy status, or a death assertion</li>     </ul>     <p>     <sup>†</sup>The boundaries between clinical findings and disorders remains a challenge in medical ontology. <div>     <a name="bnr">     </a>          <p> At its core, <Link to="/docs/api/fhir/resources/observation">Observation</Link> allows expressing a name-value pair or structured collection of name-value pairs. As such, it can support conveying any type of information desired. However, that is not its intent. <Link to="/docs/api/fhir/resources/observation">Observation</Link> is intended for capturing measurements and subjective point-in-time assessments. It is not intended to be used for those specific contexts and use cases already covered by other FHIR resources.   For example, the <a href="/docs/api/fhir/resources/allergyintolerance">AllergyIntolerance</a> resource represents a patient allergies, <a href="/docs/api/fhir/resources/medicationstatement">MedicationStatement</a> resource: medications taken by a patient, <a href="/docs/api/fhir/resources/familymemberhistory"> <Link to="/docs/api/fhir/resources/familymemberhistory">FamilyMemberHistory</Link></a> resource: a patient's family history, <a href="/docs/api/fhir/resources/procedure">Procedure</a> resource: information about a procedure, and <a href="/docs/api/fhir/resources/questionnaireresponse">QuestionnaireResponse</a> resource: a set of answers to a set of questions. The <Link to="/docs/api/fhir/resources/observation">Observation</Link> resource should not be used to record clinical diagnosis about a patient or subject that are typically captured in the <a href="/docs/api/fhir/resources/condition">Condition</a> resource or the <Link to="/docs/api/fhir/resources/clinicalimpression">ClinicalImpression</Link> resource.  The <Link to="/docs/api/fhir/resources/observation">Observation</Link> resource is often referenced by the <Link to="/docs/api/fhir/resources/condition">Condition</Link> resource to provide specific subjective and objective data to support its assertions.  There will however be situations of overlap. For example, a response to a question of "have you ever taken illicit drugs" could in principle be represented using <Link to="/docs/api/fhir/resources/medicationstatement">MedicationStatement</Link>, but most systems would treat such an assertion as an <Link to="/docs/api/fhir/resources/observation">Observation</Link>.  In some cases, such as when source data is coming from an <a href="http://www.hl7.org/implement/standards/product_brief.cfm?product_id=185">HL7 v2 </a> feed, a system might not have information that allows it to distinguish diagnosis, allergy and other "specialized" types of observations from laboratory, vital sign and other observation types intended to be conveyed with this resource.  In those circumstances, such specialized observations may also appear using this resource. Adhering to such convention is an appropriate use of <Link to="/docs/api/fhir/resources/observation">Observation</Link>. If implementers are uncertain whether a proposed use of <Link to="/docs/api/fhir/resources/observation">Observation</Link> is appropriate, they're encouraged to consult with implementers on <a href="https://chat.fhir.org/"> chat.fhir.org implementer's stream </a>     </p>   <p>   The <a href="/docs/api/fhir/resources/media">Media</a> resource captures a specific type of observation whose value is audio, video or image data. This resource is used instead of <Link to="/docs/api/fhir/resources/observation">Observation</Link> to represent such forms of information as it exposes the metadata relevant for interpreting the information.   See <Link to="/docs/api/fhir/resources/media">Media</Link>'s <a href="/docs/api/fhir/resources/media">boundaries section</a> to see how <Link to="/docs/api/fhir/resources/media">Media</Link> (and <Link to="/docs/api/fhir/resources/observation">Observation</Link>) differs from <a href="/docs/api/fhir/resources/imagingstudy">ImagingStudy</a> and <a href="/docs/api/fhir/resources/documentreference">DocumentReference</a>.   </p>   <p>   In contrast to the <Link to="/docs/api/fhir/resources/observation">Observation</Link> resource, the <a href="/docs/api/fhir/resources/diagnosticreport">DiagnosticReport</a> resource typically includes additional clinical context and some mix of atomic results, images, imaging reports, textual and coded interpretation, and formatted representations.  Laboratory reports, pathology reports, and imaging reports should be represented using the <Link to="/docs/api/fhir/resources/diagnosticreport">DiagnosticReport</Link> resource. The <Link to="/docs/api/fhir/resources/observation">Observation</Link> resource is referenced by the <Link to="/docs/api/fhir/resources/diagnosticreport">DiagnosticReport</Link> to provide the atomic results for a particular investigation.  "Laboratories routinely have a variable that is summative across a series of discrete variables - these are usually called 'impressions' or 'interpretations'. Sometimes they are algorithmically specified and sometimes they have the imprimatur of pathologists and they are conveyed in <Link to="/docs/api/fhir/resources/observation">Observation</Link> or <Link to="/docs/api/fhir/resources/diagnosticreport">DiagnosticReport</Link> instead of the <a href="/docs/api/fhir/resources/clinicalimpression">Clinical Impression</a> resource.  The <Link to="/docs/api/fhir/resources/observation">Observation</Link> resource should not be used to record clinical diagnosis about a patient or subject as discussed above.   </p>   </div>
