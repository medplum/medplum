# Observation

Measurements and simple assertions made about a patient, device or other subject.

  <Tabs queryString="intro-section">
  <TabItem value="usage" label="Usage" default>
    <div>          <p>This resource is an <a href="https://www.hl7.org/fhir/workflow.html#event">         <em>event resource</em>       </a> from a FHIR workflow perspective - see <a href="https://www.hl7.org/fhir/workflow.html">Workflow</a>.</p>     <p>Observations are a central element in healthcare, used to support diagnosis, monitor progress, determine baselines and patterns and even capture demographic characteristics.  Most observations are simple name/value pair assertions with some metadata, but some observations group other observations together logically, or even are multi-component observations. Note that the  <a href="/docs/api/fhir/resources/diagnosticreport">DiagnosticReport</a> resource provides a clinical or workflow context for a set of observations and the <a href="/docs/api/fhir/resources/observation">Observation</a> resource is referenced by <a href="/docs/api/fhir/resources/diagnosticreport">DiagnosticReport</a> to represent laboratory, imaging, and other clinical and diagnostic data to form a complete report. </p>     <p>Uses for the <a href="/docs/api/fhir/resources/observation">Observation</a> resource include:</p>     <ul>       <li>Vital signs such as <a href="https://www.hl7.org/fhir/observation-example.html">body weight</a>,  <a href="https://www.hl7.org/fhir/observation-example-bloodpressure.html">blood pressure</a>, and <a href="https://www.hl7.org/fhir/observation-example-f202-temperature.html">temperature</a>       </li>       <li>Laboratory Data like <a href="https://www.hl7.org/fhir/observation-example-f001-glucose.html">blood glucose</a>, or an <a href="https://www.hl7.org/fhir/observation-example-f205-egfr.html">estimated GFR</a>       </li>       <li>Imaging results like  <a href="https://www.hl7.org/fhir/observation-example-bmd.html">bone density</a> or fetal measurements</li>       <li>Clinical Findings<sup>*</sup> such as  <a href="https://www.hl7.org/fhir/observation-example-abdo-tender.html">abdominal tenderness</a>       </li>       <li>Device measurements such as  <a href="https://www.hl7.org/fhir/observation-example-sample-data.html">EKG data</a> or  <a href="https://www.hl7.org/fhir/observation-example-satO2.html">Pulse Oximetry data</a>       </li>       <li>Clinical assessment tools such as <a href="https://www.hl7.org/fhir/observation-example-5minute-apgar-score.html">APGAR</a> or a <a href="https://www.hl7.org/fhir/observation-example-glasgow.html">Glasgow Coma Score</a>       </li>       <li>Personal characteristics: such as  <a href="https://www.hl7.org/fhir/observation-example-eye-color.html">eye-color</a>       </li>       <li>Social history like tobacco use,  family support, or cognitive status</li>       <li>Core characteristics like pregnancy status, or a death assertion</li>     </ul>      <p>     <sup>*</sup>The boundaries between clinical findings and disorders remains a challenge in medical ontology.  Refer the <a href="https://www.hl7.org/fhir/#bnr">Boundaries</a> section below and in <a href="/docs/api/fhir/resources/condition">Condition</a> for general guidance.  These boundaries can be clarified by profiling <a href="/docs/api/fhir/resources/observation">Observation</a> for a particular use case.     </p>     <a name="core"></a>     <h3 class="self-link-parent"> Core Profiles for <a href="/docs/api/fhir/resources/observation">Observation</a> <a href="https://www.hl7.org/fhir/versions.html#std-process" title="Standards Status = Trial Use">Trial Use</a> <a href="/docs/api/fhir/resources/observation" title="link to here" class="self-link"></a></h3>     <p>The following core <a href="https://www.hl7.org/fhir/profiling.html">profiles</a> for the <a href="/docs/api/fhir/resources/observation">Observation</a> resource have been defined as well. If implementations use this Resource when expressing the profile-specific concepts as structured data, they <strong>SHALL</strong> conform to the following profiles:</p>     <table class="grid">       <thead>         <tr>           <th>Profile</th>           <th>Description</th>         </tr>       </thead>       <tbody>         <tr>           <td>             <a href="https://www.hl7.org/fhir/observation-vitalsigns.html">Vital signs</a>           </td>           <td>The FHIR Vital Signs profile sets minimum expectations for the <a href="/docs/api/fhir/resources/observation">Observation</a> Resource to record, search and fetch the vital signs (e.g. temperature, blood pressure, respiration rate, etc.) associated with a patient </td>         </tr>       </tbody>     </table>   </div>

  <TabItem value="relationships" label="Relationships">
    <div>     <a name="bnr">     </a>          <p> At its core, <a href="/docs/api/fhir/resources/observation">Observation</a> allows expressing a name-value pair or structured collection of name-value pairs. As such, it can support conveying any type of information desired. However, that is not its intent. <a href="/docs/api/fhir/resources/observation">Observation</a> is intended for capturing measurements and subjective point-in-time assessments. It is not intended to be used for those specific contexts and use cases already covered by other FHIR resources.   For example, the <a href="/docs/api/fhir/resources/allergyintolerance">AllergyIntolerance</a> resource represents a patient allergies, <a href="/docs/api/fhir/resources/medicationstatement">MedicationStatement</a> resource: medications taken by a patient, <a href="/docs/api/fhir/resources/familymemberhistory"> <a href="/docs/api/fhir/resources/familymemberhistory">FamilyMemberHistory</a></a> resource: a patient's family history, <a href="/docs/api/fhir/resources/procedure">Procedure</a> resource: information about a procedure, and <a href="/docs/api/fhir/resources/questionnaireresponse">QuestionnaireResponse</a> resource: a set of answers to a set of questions. The <a href="/docs/api/fhir/resources/observation">Observation</a> resource should not be used to record clinical diagnosis about a patient or subject that are typically captured in the <a href="/docs/api/fhir/resources/condition">Condition</a> resource or the <a href="/docs/api/fhir/resources/clinicalimpression">ClinicalImpression</a> resource.  The <a href="/docs/api/fhir/resources/observation">Observation</a> resource is often referenced by the <a href="/docs/api/fhir/resources/condition">Condition</a> resource to provide specific subjective and objective data to support its assertions.  There will however be situations of overlap. For example, a response to a question of "have you ever taken illicit drugs" could in principle be represented using <a href="/docs/api/fhir/resources/medicationstatement">MedicationStatement</a>, but most systems would treat such an assertion as an <a href="/docs/api/fhir/resources/observation">Observation</a>.  In some cases, such as when source data is coming from an <a href="http://www.hl7.org/implement/standards/product_brief.cfm?product_id=185">HL7 v2 </a> feed, a system might not have information that allows it to distinguish diagnosis, allergy and other "specialized" types of observations from laboratory, vital sign and other observation types intended to be conveyed with this resource.  In those circumstances, such specialized observations may also appear using this resource. Adhering to such convention is an appropriate use of <a href="/docs/api/fhir/resources/observation">Observation</a>. If implementers are uncertain whether a proposed use of <a href="/docs/api/fhir/resources/observation">Observation</a> is appropriate, they're encouraged to consult with implementers on <a href="https://chat.fhir.org/"> chat.fhir.org implementer's stream </a>     </p>   <p>   The <a href="/docs/api/fhir/resources/media">Media</a> resource captures a specific type of observation whose value is audio, video or image data. This resource is used instead of <a href="/docs/api/fhir/resources/observation">Observation</a> to represent such forms of information as it exposes the metadata relevant for interpreting the information.   See <a href="/docs/api/fhir/resources/media">Media</a>'s <a href="/docs/api/fhir/resources/media">boundaries section</a> to see how <a href="/docs/api/fhir/resources/media">Media</a> (and <a href="/docs/api/fhir/resources/observation">Observation</a>) differs from <a href="/docs/api/fhir/resources/imagingstudy">ImagingStudy</a> and <a href="/docs/api/fhir/resources/documentreference">DocumentReference</a>.   </p>   <p>   In contrast to the <a href="/docs/api/fhir/resources/observation">Observation</a> resource, the <a href="/docs/api/fhir/resources/diagnosticreport">DiagnosticReport</a> resource typically includes additional clinical context and some mix of atomic results, images, imaging reports, textual and coded interpretation, and formatted representations.  Laboratory reports, pathology reports, and imaging reports should be represented using the <a href="/docs/api/fhir/resources/diagnosticreport">DiagnosticReport</a> resource. The <a href="/docs/api/fhir/resources/observation">Observation</a> resource is referenced by the <a href="/docs/api/fhir/resources/diagnosticreport">DiagnosticReport</a> to provide the atomic results for a particular investigation.  "Laboratories routinely have a variable that is summative across a series of discrete variables - these are usually called 'impressions' or 'interpretations'. Sometimes they are algorithmically specified and sometimes they have the imprimatur of pathologists and they are conveyed in <a href="/docs/api/fhir/resources/observation">Observation</a> or <a href="/docs/api/fhir/resources/diagnosticreport">DiagnosticReport</a> instead of the <a href="/docs/api/fhir/resources/clinicalimpression">Clinical Impression</a> resource.  The <a href="/docs/api/fhir/resources/observation">Observation</a> resource should not be used to record clinical diagnosis about a patient or subject as discussed above.   </p>   </div>
