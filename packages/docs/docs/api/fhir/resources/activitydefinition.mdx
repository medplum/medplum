# ActivityDefinition

This resource allows for the definition of some activity to be performed, independent of a particular patient, practitioner, or other performance context.

<div> <a name="bnc"></a> <p><strong>This resource is a <em>definition</em> resource from a FHIR workflow perspective</strong> - see <a href="https://www.hl7.org/fhir/workflow.html">Workflow</a>, specifically <a href="https://www.hl7.org/fhir/workflow.html#definition">Definition</a>.</p> <p>An <Link to="/docs/api/fhir/resources/activitydefinition">ActivityDefinition</Link> is a shareable, consumable description of some activity to be performed. It may be used to specify actions to be taken as part of a workflow, order set, or protocol, or it may be used independently as part of a catalog of activities such as orderables.</p> <p>For more information on how activity definitions can be used to construct request resources, see the Applying an <Link to="/docs/api/fhir/resources/activitydefinition">ActivityDefinition</Link> topic below.</p> </div>
  </TabItem>
  <TabItem value="relationships" label="Relationships">
<div> <p>In general, an activity definition is simply a conceptual description of some specific action that should be taken. An instance of an <Link to="/docs/api/fhir/resources/activitydefinition">ActivityDefinition</Link> does not indicate that any action has been performed (as an <a href="https://www.hl7.org/fhir/workflow.html#event">event</a> resource does), nor does it indicate the actual <i>intent</i> to carry out any particular action (as a <a href="https://www.hl7.org/fhir/workflow.html#request">request</a> resource does). Instead, an activity definition provides a reusable template that can be used to construct specific <i>request</i> resources such as <a href="/docs/api/fhir/resources/servicerequest">ServiceRequest</a> and <a href="/docs/api/fhir/resources/medicationrequest">MedicationRequest</a>. </p> <p>Note that this is conceptually similar to the <a href="/docs/api/fhir/resources/task">Task</a> resource as well, with the distinction being that <Link to="/docs/api/fhir/resources/activitydefinition">ActivityDefinition</Link> represents the description of a task in the abstract, while the <Link to="/docs/api/fhir/resources/task">Task</Link> resource is used to track a specific instance of a task as it moves through the steps of a workflow.</p> </div>
