    <div> <a name="scope"></a>   <p> The <a href="/docs/api/fhir/resources/testscript">TestScript</a> resource is used to define tests that can be executed on one or more FHIR servers. The <a href="/docs/api/fhir/resources/testreport">TestReport</a> resource defines how systems should encode the summarized results of executing a <a href="/docs/api/fhir/resources/testscript">TestScript</a>. </p> <p> The <a href="/docs/api/fhir/resources/testreport">TestReport</a> structure mirrors the <a href="/docs/api/fhir/resources/testscript">TestScript</a> concepts of having sections for setup, tests, and teardown. If present, these ordered lists should mirror the actions (either operations or assertions) of the referenced <a href="/docs/api/fhir/resources/testscript">TestScript</a> with a result code: pass, skip, fail, warning, or error. </p> </div>
