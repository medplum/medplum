<div> <a name="scope"></a> <p> The <a href="/docs/api/fhir/resources/testscript">TestScript</a> resource is used to define tests that can be executed on one or more FHIR servers. The <Link to="/docs/api/fhir/resources/testreport">TestReport</Link> resource defines how systems should encode the summarized results of executing a <Link to="/docs/api/fhir/resources/testscript">TestScript</Link>. </p> <p> The <Link to="/docs/api/fhir/resources/testreport">TestReport</Link> structure mirrors the <Link to="/docs/api/fhir/resources/testscript">TestScript</Link> concepts of having sections for setup, tests, and teardown. If present, these ordered lists should mirror the actions (either operations or assertions) of the referenced <Link to="/docs/api/fhir/resources/testscript">TestScript</Link> with a result code: pass, skip, fail, warning, or error. </p> </div>
