<div> <a name="scope"></a> <p> The <Link to="/docs/api/fhir/resources/healthcareservice">HealthcareService</Link> resource is used to describe a single healthcare service or category of services that are provided by an organization at a location.<br/> The location of the services could be virtual, as with telemedicine services.<br/> </p> <p>Common examples of <Link to="/docs/api/fhir/resources/healthcareservice">HealthcareServices</Link> resources are:</p> <ul> <li>Allied Health</li> <li>Clinical Neuropsychologist</li> <li>Podiatry Service</li> <li>Smallville Hospital Emergency Services</li> <li>Respite care provided at a nursing home or hostel</li> <li>24hr crisis telephone counseling service</li> <li>Information, advice and/or referral services; Disability, Telecommunications</li> <li>Rural TeleHealth Services</li> <li>Hospital in the home</li> <li>Yellow Cabs</li> <li>Pharmacy</li> <li>Active Rehab</li> <li>Social Support</li> <li>Drug and/or alcohol counseling</li> <li>Day Programs, Adult Training &amp; Support Services</li> <li>Consulting psychologists and/or psychology services</li> <li>Group Hydrotherapy</li> <li>Little River Home Maintenance</li> </ul> <p> <i>HealthcareService resources do not represent Computer related Services (not SOA)</i> </p> <p>Example uses of <Link to="/docs/api/fhir/resources/healthcareservice">HealthcareService</Link> resources are:</p> <ul> <li>National Services Directory - Consumer Focus</li> <li>National Services Directory - <Link to="/docs/api/fhir/resources/practitioner">Practitioner</Link> Referrals Searching</li> <li>Organization's Client Portal - to locate services / book appointments</li> <li> <Link to="/docs/api/fhir/datatypes/address">Address</Link> book of services for Referrals<br/> <i>including references to <Link to="/docs/api/fhir/resources/questionnaire">Questionnaires</Link> for assessments that are required as part of the referral</i> </li> <li> Health Network internal directory <i> Used for tracking available services offered internally, and also those offered by business partners.<br/> This information may also include costing information. </i> </li> </ul> </div>
  </TabItem>
  <TabItem value="relationships" label="Relationships">
<div> <a name="bnr"></a> <p> The <Link to="/docs/api/fhir/resources/healthcareservice">HealthcareService</Link> resource can be used with the <Link to="/docs/api/fhir/resources/schedule">Schedule</Link> resource to define actual availability of the service. This would be done by using the <Link to="/docs/api/fhir/resources/schedule">Schedule</Link>'s Actor property. </p> <p> When creating an <Link to="/docs/api/fhir/resources/appointment">Appointment</Link>, the <Link to="/docs/api/fhir/resources/healthcareservice">HealthcareService</Link> is to be assigned to one of the participants.<br/> It is up to the scheduling system to determine if the service is available and can be accepted. </p> <p>The <Link to="/docs/api/fhir/resources/healthcareservice">HealthcareService</Link> resource is used with the following resources:</p> <ul> <li><b><a href="/docs/api/fhir/resources/organization">Organization</a></b>: The organization provides the services, the healthcareservice describes the services</li> <li><b><a href="/docs/api/fhir/resources/servicerequest">ServiceRequest</a></b>: This is a subject specific request for a specific service, it may be to a specific healthcare service, and/or simply a coded service value. The granularity included in the request codes may be finer than defined in the healthcare service. </li> <li><b><a href="/docs/api/fhir/resources/organizationaffiliation">OrganizationAffiliation</a></b>: Might constrain a list of healthcare services that are available between 2 organizations</li> <li><b><a href="/docs/api/fhir/resources/location">Location</a></b>: Specifies the place where the service(s) are offered/available within.</li> </ul> </div>
