# TestScript

A structured set of tests against a FHIR server or client implementation to determine compliance against the FHIR specification.

  <TabItem value="usage" label="Usage">
    <div> <a name="scope"></a>   <p> The <a href="/docs/api/fhir/resources/testscript">TestScript</a> resource is used to define tests that can be executed on one or more FHIR servers. The <a href="/docs/api/fhir/resources/testscript">TestScript</a> resource would typically contain </p><ul>   <li>a list of fixtures (required resources used in the tests)</li>   <li>setup procedures</li>   <li>a suite of thematically related tests</li>   <li>teardown procedures</li> </ul> <p></p><p> For example, one <a href="/docs/api/fhir/resources/testscript">TestScript</a> might feature a set of tests focusing on searching <a href="/docs/api/fhir/resources/patient">Patients</a> and validating the <a href="/docs/api/fhir/resources/bundle">Bundle</a> responses. The fixtures for such a test would contain a list of <a href="/docs/api/fhir/resources/patient">Patient</a> resources that are required for the test to complete successfully. The setup procedures create the fixtures on the FHIR server being tested. A series of tests execute various search parameters and search for the fixtures in the results. The teardown procedures would then clean up (delete) the fixtures on FHIR server that were created during the setup procedures. </p><p> The purpose of the <a href="/docs/api/fhir/resources/testscript">TestScript</a> is to encode in an executable representation tests that can be used to </p><ol>   <li>determine whether a given FHIR server adheres to the FHIR specification and</li>   <li>determine whether two or more FHIR servers implement capabilities in a compatible or interoperable manner.</li> </ol> It might not be possible to fully automate the latter goal (especially with regards to business rules that might ride on top of different implementations), however the tests should be able to determine whether two servers support the operations, value sets, profiles, and extensions to exchange <a href="/docs/api/fhir/resources/bundle">Bundles</a> of Resources (such as returned from the <a href="https://www.hl7.org/fhir/operation-patient-everything.html">Patient $everything operation</a>).    <p></p> <p>  </p> </div>

    <div> <a name="bnr"></a>  <p> The <a href="/docs/api/fhir/resources/testscript">TestScript</a> resource should <b>NOT</b> be used to represent Clinical tests, Prescriptions, or any other Healthcare related concept. The <a href="/docs/api/fhir/resources/testscript">TestScript</a> resource is an infrastructure support resource used to represent standardized tests to determine an implementation's level of adherence to the FHIR specification. </p> <p> <a href="/docs/api/fhir/resources/testscript">TestScript</a> is a part of the conformance framework and is used to validate the behavior of FHIR systems, specifically their correct implementation of <a href="/docs/api/fhir/resources/structuredefinition">StructureDefinition</a>, <a href="/docs/api/fhir/resources/valueset">ValueSet</a>, <a href="/docs/api/fhir/resources/operationdefinition">OperationDefinition</a>, <a href="/docs/api/fhir/resources/capabilitystatement">CapabilityStatement</a> and other FHIR resources that govern system behavior. <a href="/docs/api/fhir/resources/testscript">TestScript</a> instances may be included as part of <a href="/docs/api/fhir/resources/implementationguide">ImplementationGuides</a> to help define and test the desired behavior of systems that choose to comply with the implementation guide. </p> <p> The following resources represent the FHIR conformance framework that are used to express the expected behavior of a FHIR compliant system: </p> <ul>   <li><a href="/docs/api/fhir/resources/valueset" title="A value set specifies a set of codes drawn from one or more code systems.">ValueSet</a> </li>   <li><a href="/docs/api/fhir/resources/conceptmap" title="A statement of relationships from one set of concepts to one or more other concepts - either concepts in code systems, or data element/data element concepts, or classes in class models.">ConceptMap</a> </li>   <li><a href="/docs/api/fhir/resources/structuredefinition" title="A definition of a FHIR structure. This resource is used to gather all the parts of an implementation guide into a logical whole, and to publish a computable definition of all the parts.">ImplementationGuide</a> </li> </ul>  </div>
