# TestScript

A structured set of tests against a FHIR server or client implementation to determine compliance against the FHIR specification.

  <TabItem value="usage" label="Usage">
<div> <a name="scope"></a> <p> The <Link to="/docs/api/fhir/resources/testscript">TestScript</Link> resource is used to define tests that can be executed on one or more FHIR servers. The <Link to="/docs/api/fhir/resources/testscript">TestScript</Link> resource would typically contain </p><ul>   <li>a list of fixtures (required resources used in the tests)</li>   <li>setup procedures</li>   <li>a suite of thematically related tests</li>   <li>teardown procedures</li> </ul> <p></p><p> For example, one <Link to="/docs/api/fhir/resources/testscript">TestScript</Link> might feature a set of tests focusing on searching <Link to="/docs/api/fhir/resources/patient">Patients</Link> and validating the <Link to="/docs/api/fhir/resources/bundle">Bundle</Link> responses. The fixtures for such a test would contain a list of <Link to="/docs/api/fhir/resources/patient">Patient</Link> resources that are required for the test to complete successfully. The setup procedures create the fixtures on the FHIR server being tested. A series of tests execute various search parameters and search for the fixtures in the results. The teardown procedures would then clean up (delete) the fixtures on FHIR server that were created during the setup procedures. </p><p> The purpose of the <Link to="/docs/api/fhir/resources/testscript">TestScript</Link> is to encode in an executable representation tests that can be used to </p><ol>   <li>determine whether a given FHIR server adheres to the FHIR specification and</li>   <li>determine whether two or more FHIR servers implement capabilities in a compatible or interoperable manner.</li> </ol> It might not be possible to fully automate the latter goal (especially with regards to business rules that might ride on top of different implementations), however the tests should be able to determine whether two servers support the operations, value sets, profiles, and extensions to exchange <Link to="/docs/api/fhir/resources/bundle">Bundles</Link> of Resources (such as returned from the <a href="https://www.hl7.org/fhir/operation-patient-everything.html">Patient $everything operation</a>).    <p></p> <p> </p> </div>

<div> <a name="bnr"></a> <p> The <Link to="/docs/api/fhir/resources/testscript">TestScript</Link> resource should <b>NOT</b> be used to represent Clinical tests, Prescriptions, or any other Healthcare related concept. The <Link to="/docs/api/fhir/resources/testscript">TestScript</Link> resource is an infrastructure support resource used to represent standardized tests to determine an implementation's level of adherence to the FHIR specification. </p> <p> <Link to="/docs/api/fhir/resources/testscript">TestScript</Link> is a part of the conformance framework and is used to validate the behavior of FHIR systems, specifically their correct implementation of <Link to="/docs/api/fhir/resources/structuredefinition">StructureDefinition</Link>, <Link to="/docs/api/fhir/resources/valueset">ValueSet</Link>, <Link to="/docs/api/fhir/resources/operationdefinition">OperationDefinition</Link>, <Link to="/docs/api/fhir/resources/capabilitystatement">CapabilityStatement</Link> and other FHIR resources that govern system behavior. <Link to="/docs/api/fhir/resources/testscript">TestScript</Link> instances may be included as part of <Link to="/docs/api/fhir/resources/implementationguide">ImplementationGuides</Link> to help define and test the desired behavior of systems that choose to comply with the implementation guide. </p> <p> The following resources represent the FHIR conformance framework that are used to express the expected behavior of a FHIR compliant system: </p> <ul>   <li><a href="/docs/api/fhir/resources/valueset" title="A value set specifies a set of codes drawn from one or more code systems.">ValueSet</a> </li>   <li><a href="/docs/api/fhir/resources/conceptmap" title="A statement of relationships from one set of concepts to one or more other concepts - either concepts in code systems, or data element/data element concepts, or classes in class models.">ConceptMap</a> </li>   <li><a href="/docs/api/fhir/resources/structuredefinition" title="A definition of a FHIR structure. This resource is used to describe the underlying resources, data types defined in FHIR, and also for describing extensions, and constraints on resources and data types.">StructureDefinition</a> </li>   <li><a href="/docs/api/fhir/resources/capabilitystatement" title="A capability statement is a set of capabilities of a FHIR Server that may be used as a statement of actual server functionality or a statement of required or desired server implementation.">CapabilityStatement</a> </li>   <li><a href="/docs/api/fhir/resources/operationdefinition" title="A formal computable definition of an operation (on the RESTful interface) or a named query (using the search interaction).">OperationDefinition</a> </li>   <li><a href="/docs/api/fhir/resources/searchparameter" title="A Search Parameter that defines a named search item that can be used to search/filter on a resource.">SearchParameter</a> </li>   <li><a href="/docs/api/fhir/resources/implementationguide" title="A set of rules or how FHIR is used to solve a particular problem. This resource is used to gather all the parts of an implementation guide into a logical whole, and to publish a computable definition of all the parts.">ImplementationGuide</a> </li> </ul> </div>

<div> <a name="bnc"></a> <h3 class="self-link-parent"> Background <a href="https://www.hl7.org/fhir/testscript.html#7.3.3.1" title="link to here" class="self-link"></a></h3> <p>   The <Link to="/docs/api/fhir/resources/testscript">TestScript</Link> resource is designed to establish testing as a first class artifact within the FHIR specification.  This resource allows defining a suite of tests that can be executed on one or more FHIR servers and clients. </p> <p> Implementers should be familiar with the testing concepts and descriptions found in <a href="https://www.hl7.org/fhir/testing.html">Testing FHIR</a> before working with this resource. </p> </div>
