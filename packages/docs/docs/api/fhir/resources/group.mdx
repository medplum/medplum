# Group

Represents a defined collection of entities that may be discussed or acted upon collectively but which are not expected to act collectively, and are not formally or legally recognized; i.e. a collection of entities that isn't an <Link to="/docs/api/fhir/resources/organization">Organization</Link>.

<div> <a name="scope"></a> <a name="use"></a> <h3 class="self-link-parent"> Use Cases <a href="/docs/api/fhir/resources/group" title="link to here" class="self-link"></a></h3> <p>The <Link to="/docs/api/fhir/resources/group">Group</Link> resource is used in one of two ways:</p> <ol>   <li>To define a group of specific people, animals, devices, etc. that is being tracked, examined or otherwise referenced as part of healthcare-related activities</li>   <li>To define a set of <i>possible</i> people, animals, devices, etc. that are of interest for some intended future healthcare-related activities</li> </ol> <p>Examples of the former could include group therapy or treatment sessions, exposed entities tracked as part of public health, etc.  The latter might be used to define expected subjects for a clinical study.</p> <p>Both use cases are handled by a single resource because the data elements captured tend to be similar.</p> </div>

<div> <a name="bnr"></a> <p> There are a number of mechanisms in FHIR for communicating collections of resources: </p> <ul> <li> The <a href="/docs/api/fhir/resources/list">List</a> resource - enumerates a flat collection of resources and provides features for managing the collection.   While a particular <Link to="/docs/api/fhir/resources/list">List</Link> instance may represent a "snapshot", from a business process perspective the notion of "List"  is dynamic â€“ items are added and removed over time.  The <Link to="/docs/api/fhir/resources/list">List</Link> resource references other resources.  <Link to="/docs/api/fhir/resources/list">Lists</Link> may be  curated and have specific business meaning. </li> <li> This <Link to="/docs/api/fhir/resources/group">Group</Link> resource - defines a group of specific people, animals, devices etc. by enumerating them,  or by describing qualities that group members have. The group resource refers to other resources, possibly implicitly.   <Link to="/docs/api/fhir/resources/group">Groups</Link> are intended to be acted upon or observed as a whole; e.g. performing therapy on a group, calculating risk for a group,  etc.  This resource will commonly be used for public health (e.g. describing an at-risk population), clinical trials (e.g.  defining a test subject pool) and similar purposes. </li> <li><a href="/docs/api/fhir/resources/careteam">CareTeam</a>. <Link to="/docs/api/fhir/resources/group">Group</Link> is distinct from <Link to="/docs/api/fhir/resources/careteam">CareTeam</Link>. <Link to="/docs/api/fhir/resources/group">Group</Link> is patient-independent and identifies an  undifferentiated set of individuals who are intended to be the target of one or more clinical activities (e.g. set of  clinical trial participants, set of individuals impacted by or at risk of a public health event, a herd or flock, etc.)  The <Link to="/docs/api/fhir/resources/careteam">CareTeam</Link> resource establishes a set of relationships and roles and is specific to a particular <Link to="/docs/api/fhir/resources/patient">Patient</Link>.  The actors are the individual members or organized group of individuals. <Link to="/docs/api/fhir/resources/careteam">CareTeam</Link> can be referenced by <Link to="/docs/api/fhir/resources/episodeofcare">EpisodeOfCare</Link>,  <Link to="/docs/api/fhir/resources/encounter">Encounter</Link>, or <Link to="/docs/api/fhir/resources/careplan">CarePlan</Link> to identify the set of individuals (and their respective roles) who are intended to be involved  in providing the care defined by those resources. </li> <li> The <a href="/docs/api/fhir/resources/bundle">Bundle</a> resource - is an infrastructure container for a group of resources.  It does not have narrative  and is used to group collections of resources for transmission, persistence or processing (e.g. messages, documents, transactions,  query responses, etc.)  The content of bundles is typically algorithmically determined for a particular exchange or persistence purpose. </li> <li> The <a href="/docs/api/fhir/resources/composition">Composition</a> resource - defines a set of healthcare-related information that is assembled  together into a single logical document that provides a single coherent statement of meaning, establishes its own context and  that has clinical attestation with regard to who is making the statement. The composition resource provides the basic structure  of a FHIR <a href="https://www.hl7.org/fhir/documents.html">document</a>.  The full content of the document is expressed using a bundle.  <Link to="/docs/api/fhir/resources/composition">Compositions</Link> will  often reference <Link to="/docs/api/fhir/resources/list">Lists</Link> as the focus of particular sections. </li> <li> The <a href="https://www.hl7.org/fhir/domainresource.html">DomainResource</a>.<code>contained</code> element - allows multiple resources to be nested inside any DomainResource.  This is a special type of grouping where the grouped resources lose independent existence - they no longer have their own identifiers, can't easily be queried independently, etc.  Use of this grouping is a technical  mechanism for managing the independence of resources and has no impact on meaning.   </li> </ul> Contained, bundled and remotely referenced resources convey the same meaning. </div>
