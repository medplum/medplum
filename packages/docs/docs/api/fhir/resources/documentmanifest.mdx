    <div> <a name="scope"></a>  <p>  A document manifest gathers a set of Resources  into a single package with some clinical context that may be the subject of workflow such as targeted delivery.  The usage of "document" in the case of <a href="/docs/api/fhir/resources/documentmanifest">DocumentManifest</a>, covers the broader scope of "document" as defined in REST and HTTP.  This use is not limited to the formal "Document" model used for <a href="https://www.hl7.org/fhir/documents.html">FHIR documents</a> or <a href="/docs/api/fhir/resources/documentreference">DocumentReference</a> resources. </p> <p> Typically, <a href="/docs/api/fhir/resources/documentmanifest">DocumentManifest</a> resources are used in document indexing systems, such as <a href="http://wiki.ihe.net/index.php?title=Cross-Enterprise_Document_Sharing">IHE XDS </a>. Such as profiled in <a href="http://wiki.ihe.net/index.php/Mobile_access_to_Health_Documents_(MHD)">IHE Mobile access to Health Documents </a>. When used as a Manifest of "Documents" in this way, the <a href="/docs/api/fhir/resources/documentmanifest">DocumentManifest</a> will point exclusively to <a href="/docs/api/fhir/resources/documentreference">DocumentReference</a> resources. </p> <p> A <a href="/docs/api/fhir/resources/documentmanifest">DocumentManifest</a> resource could be used to bring clinical context grouping to a broader set of Resources.</p> </div>
