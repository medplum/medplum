# RequestGroup

A group of related requests that can be used to capture intended activities that have inter-dependencies such as "give this medication after that one".

    <div> <a name="bnc"></a>  <p><strong>This resource is a <em>request</em> resource from a FHIR workflow perspective</strong> - see <a href="https://www.hl7.org/fhir/workflow.html">Workflow</a>, specifically <a href="https://www.hl7.org/fhir/workflow.html#request">Request</a>.</p>  <p>The <a href="/docs/api/fhir/resources/requestgroup">RequestGroup</a> resource is used to represent a group of optional activities that may be performed for a specific patient or context. This resource is often, but not always, the result of applying a specific <a href="/docs/api/fhir/resources/plandefinition">PlanDefinition</a> to a particular patient. Other than differences that tie the <a href="/docs/api/fhir/resources/requestgroup">RequestGroup</a> to a particular subject and setting, the <code>actionDefinition</code> element of <a href="/docs/api/fhir/resources/plandefinition">PlanDefinition</a> is identical to the <code>action</code> element of the <a href="/docs/api/fhir/resources/requestgroup">RequestGroup</a>, allowing the same features and functionality to be used in both places to describe optionality of and relationships between activities in a workflow.</p>  <p>RequestGroups can contain hierarchical groups of actions, where each specific action references the action to be performed (in terms of a <a href="https://www.hl7.org/fhir/workflow.html#request">Request</a> resource), and each group describes additional behavior, relationships, and applicable conditions between the actions in the overall group.</p>  </div>
