# GDPR Consent Requirements

## Purpose

This compliance rule ensures that personal data collection and processing
complies with the General Data Protection Regulation (GDPR) consent requirements.

## Consent Principles

### Freely Given

- Consent must not be a precondition for accessing a service
- Users must have genuine choice
- No bundled consent for unrelated processing

### Specific

- Consent must be specific to each purpose
- Separate consent for different processing activities
- Clear identification of data controller

### Informed

- Clear and plain language
- Identity of the controller
- Purpose of processing
- Types of data collected
- Right to withdraw consent

### Unambiguous

- Clear affirmative action required
- No pre-ticked boxes
- Silence or inactivity does not constitute consent

## Implementation Requirements

| Requirement | Description |
|-------------|-------------|
| Consent recording | Store timestamp, version, and method of consent |
| Withdrawal mechanism | Easy withdrawal, as easy as giving consent |
| Age verification | Parental consent for minors (under 16 in most EU countries) |
| Audit trail | Complete history of consent changes |

## Data Subject Rights

Under GDPR, data subjects have the right to:

1. **Access** - Obtain confirmation and access to their data
2. **Rectification** - Correct inaccurate personal data
3. **Erasure** - Request deletion ("right to be forgotten")
4. **Restriction** - Limit processing of their data
5. **Portability** - Receive data in machine-readable format
6. **Object** - Object to processing for specific purposes
